## 🚀 התחלה מהירה

### תחילת עבודה עם הבוט

1. **התחלה ראשונה:**
   שלח `/start` לבוט ותקבל:
   - תפריט ראשי עם כפתורים בתחתית המסך
   - הודעת ברכה אישית
   - רישום אוטומטי במערכת

2. **התפריט הראשי:**
   לאחר `/start`, תראה תפריט עם כפתורים:
   
   ```
   ┌─────────────────────┬─────────────────────┐
   │  🗜️ יצירת ZIP      │  ➕ הוסף קוד חדש   │
   ├─────────────────────┼─────────────────────┤
   │ 📚 הצג את כל        │  🔧 GitHub          │
   │    הקבצים שלי       │                     │
   ├─────────────────────┼─────────────────────┤
   │ ⚡ עיבוד Batch      │ 📥 ייבוא ZIP       │
   │                     │    מריפו            │
   ├─────────────────────┼─────────────────────┤
   │ ☁️ Google Drive     │ ℹ️ הסבר על הבוט    │
   └─────────────────────┴─────────────────────┘
   ```

3. **קבלת עזרה:**
   - לחץ על **ℹ️ הסבר על הבוט** בתפריט הראשי
   - או שלח `/help` לצפייה בעמוד עזרה אינטראקטיבי
   - ניתן לדפדף בין עמודי העזרה עם כפתורי ניווט

## 📝 פעולות בסיסיות

### שמירת קוד חדש ➕

**דרך 1 - כפתור בתפריט:**
1. לחץ על **➕ הוסף קוד חדש** בתפריט הראשי
2. בחר אופן השמירה:
   - 📝 **קטע קוד חדש** - שמירת קוד רגיל
   - 📎 **קובץ מצורף** - שמירת קובץ שתשלח
3. אם בחרת קטע קוד:
   - כתוב את שם הקובץ (למשל: `calculator.py`)
   - הוסף תגיות אם תרצה (למשל: `calculator.py #python #math`)
4. שלח את הקוד בדיוק כפי שתרצה שיישמר
5. הבוט ישמור את הקוד עם:
   - שם הקובץ
   - שפת התכנות (זיהוי אוטומטי)
   - תגיות (אם הוספת)
   - תאריך ושעה

**תגיות:**
- הוספת תגיות עם `#` אחרי שם הקובץ
- דוגמה: `api.py #python #api #rest`
- עוזר לארגן ולמצוא קבצים מהר יותר

**דוגמה מלאה:**
```
👤 [לוחץ על ➕ הוסף קוד חדש]

🤖 🎯 בחר אופן שמירה:
   [📝 קטע קוד חדש] [📎 קובץ מצורף]

👤 [לוחץ על 📝 קטע קוד חדש]

🤖 מה שם הקובץ?
   (ניתן להוסיף תגיות: filename.py #tag1 #tag2)

👤 calculator.py #python #math

🤖 מוכן לשמור את calculator.py
   🏷️ תגיות: python, math
   אנא שלח את קטע הקוד:

👤 def add(a, b):
      return a + b

   def multiply(a, b):
      return a * b

🤖 ✅ נשמר בהצלחה!
   📁 calculator.py
   🔤 שפה: python
   🏷️ תגיות: python, math
   📊 גודל: 89 תווים
```

**דרך 2 - פקודה (חלופה):**
ניתן גם להשתמש בפקודה `/save filename.py` ולאחר מכן לשלוח את הקוד.

### צפייה בקבצים 📚

**צעדים:**
1. לחץ על **📚 הצג את כל הקבצים שלי** בתפריט הראשי
2. תפריט בחירה יפתח עם אפשרויות:
   - 🔎 **חפש קובץ** - חיפוש לפי שם/שפה/תגית
   - 🗂 **לפי ריפו** - קבצים מאורגנים לפי פרויקט
   - 📂 **קבצים גדולים** - תצוגה מדורגת לקבצים ארוכים
   - 📁 **שאר הקבצים** - כל השאר
   - 📦 **קבצי ZIP** - גיבויים/ארכיונים
   - ⭐ **מועדפים** - קבצים מסומנים כמועדפים
   - 🗑️ **סל מיחזור** - קבצים שנמחקו
3. בחר את הקטגוריה הרצויה
4. רשימת קבצים תוצג (10 קבצים לעמוד)
5. לחץ על שם קובץ לפתיחת תפריט פעולות

**תפריט קובץ - כל האפשרויות:**
כשאתה לוחץ על קובץ, תראה תפריט עם כל הפעולות:

```
🎯 מרכז בקרה מתקדם

📄 קובץ: calculator.py
🧠 שפה: python

┌─────────────────┬─────────────────┐
│ 👁️ הצג קוד     │ ✏️ ערוך        │
├─────────────────┼─────────────────┤
│ 📝 שנה שם       │ 📝 ערוך הערה   │
├─────────────────┼─────────────────┤
│ 📚 היסטוריה    │ 📥 הורד         │
├─────────────────┼─────────────────┤
│ 🔗 שתף קוד     │                 │
├─────────────────┼─────────────────┤
│ 🔄 שכפול       │ 🗑️ מחק          │
└─────────────────┴─────────────────┘

[🔙 חזרה לרשימה]
```

### סטטיסטיקות 📊

**איך לראות סטטיסטיקות:**
- שלח `/stats` כדי לראות סטטיסטיקות אישיות

**מה מוצג:**
- מספר קבצים שנשמרו
- שפות תכנות בשימוש
- גודל כולל של הקבצים
- מספר התגיות
- קבצים אחרונים

## 📁 ניהול קבצים

### צפייה בקובץ 👁️

**איך:**
1. פתח את רשימת הקבצים דרך **📚 הצג את כל הקבצים שלי**
2. לחץ על שם הקובץ
3. בתפריט הקובץ, לחץ על **👁️ הצג קוד**

**מה מוצג:**
- תוכן הקובץ עם הדגשת תחביר צבעונית
- מידע על הקובץ (שפה, תאריך, גודל)
- כפתורי פעולה מתחת לקוד:
  - ✏️ **ערוך** - עריכת הקובץ
  - ⭐ **מועדף** - הוספה/הסרה ממועדפים
  - 🌐 **שתף** - שיתוף הקובץ
  - 📥 **הורד** - הורדת הקובץ
  - 📊 **ניתוח** - ניתוח מתקדם
  - 📋 **העתק** - העתקת תוכן

**לקבצים גדולים:**
- קבצים גדולים מוצגים בחלקים (100 שורות בכל פעם)
- כפתורים: **⬇️ הצג עוד** / **⬆️ הצג פחות**
- אפשרות לקפוץ לשורה ספציפית

### עריכת קובץ ✏️

**איך:**
1. פתח את תפריט הקובץ (לחץ על שם הקובץ ברשימה)
2. לחץ על **✏️ ערוך**
3. שלח את הקוד החדש (כולו)
4. הבוט ישמור את הגרסה החדשה

**מה קורה:**
- הגרסה הישנה נשמרת בהיסטוריה
- הגרסה החדשה הופכת להיות הגרסה הנוכחית
- ניתן לשחזר גרסאות קודמות דרך **📚 היסטוריה**

**הערה:** שלח את כל הקוד מחדש, לא רק את החלק שמשתנה.

### שינוי שם קובץ 📝

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **📝 שנה שם**
3. כתוב את השם החדש
4. השם יתעדכן אוטומטית

### עריכת הערה/תיאור 📝

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **📝 ערוך הערה**
3. כתוב את התיאור/הערה
4. התיאור יתעדכן

ההערה תוצג בתפריט הקובץ מתחת לשם הקובץ.

### מחיקת קובץ 🗑️

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **🗑️ מחק**
3. אשר את המחיקה

**מה קורה:**
- הקובץ מועבר לסל מיחזור
- ניתן לשחזר אותו מסל המיחזור
- מחיקה מסל המיחזור היא בלתי הפיכה

### הורדת קובץ 📥

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **📥 הורד**
3. הקובץ יישלח כ-Document שניתן להוריד

### היסטוריית גרסאות 📚

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **📚 היסטוריה**
3. רשימת כל הגרסאות תוצג

**מה תראה:**
- כל הגרסאות עם תאריך ושעה
- גודל כל גרסה
- אפשרות לצפות בכל גרסה
- אפשרות לשחזר גרסה קודמת

### שכפול קובץ 🔄

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **🔄 שכפול**
3. הקובץ יועתק עם שם חדש (copy של השם המקורי)

שכפול שימושי ליצירת גרסה חדשה מבלי לשנות את המקור.

## 🗂 ארגון ואיתור

### מועדפים ⭐

**איך לסמן כמועדף:**
1. פתח את הקובץ (לחץ על השם ברשימה)
2. לחץ על **👁️ הצג קוד** או פשוט פתח את הקובץ
3. בתצוגת הקוד, לחץ על כפתור **⭐** (מועדף)
4. הקובץ מסומן כמועדף

**צפייה במועדפים:**
1. לחץ על **📚 הצג את כל הקבצים שלי**
2. בתפריט הבחירה, לחץ על **⭐ מועדפים**
3. רשימת כל המועדפים תוצג

**הסרת מועדף:**
- לחץ שוב על כפתור **⭐** בתצוגת הקובץ

### חיפוש קבצים 🔎

**איך:**
1. לחץ על **📚 הצג את כל הקבצים שלי**
2. בתפריט, לחץ על **🔎 חפש קובץ**
3. כתוב את החיפוש שלך

**סוגי חיפוש:**
- **חיפוש בשם:** `calculator` - מחפש בשם הקובץ
- **חיפוש לפי שפה:** `lang:python` - מחפש לפי שפת תכנות
- **חיפוש לפי תגית:** `tag:api` - מחפש לפי תגית
- **חיפוש משולב:** `name:util lang:python` - חיפוש משולב
- **חיפוש בתוכן:** הבוט יחפש גם בתוכן הקוד

**דוגמאות:**
```
calculator           → מחפש קבצים עם "calculator" בשם
lang:python          → כל קבצי Python
tag:repo:owner/proj  → קבצים מפרויקט ספציפי
name:util lang:ts    → קבצי TypeScript עם "util" בשם
```

**תוצאות:**
- רשימה של קבצים תואמים
- מידע קצר על כל קובץ (שם, שפה, תאריך)
- לחץ על קובץ לעבור לתפריט הקובץ

### ארגון לפי ריפו 🗂

**איך:**
1. לחץ על **📚 הצג את כל הקבצים שלי**
2. בתפריט, לחץ על **🗂 לפי ריפו**
3. רשימת כל הריפוזיטורים תוצג
4. לחץ על ריפו כדי לראות את כל הקבצים שלו

**מה זה:**
- קבצים שיובאו מ-GitHub או סומנו עם תגית `repo:` מסודרים לפי פרויקט
- עוזר לארגן קבצים מפרויקטים שונים

**דוגמה:**
```
🗂 קבצים לפי ריפו:

repo:owner/project1 (15 קבצים)
repo:owner/project2 (8 קבצים)
repo:personal/scripts (5 קבצים)

[לחץ על ריפו לראות את הקבצים שלו]
```

### קבצים גדולים 📂

**איך:**
1. לחץ על **📚 הצג את כל הקבצים שלי**
2. בתפריט, לחץ על **📂 קבצים גדולים**
3. רשימת קבצים גדולים תוצג

**מה מוגדר כגדול:**
- קבצים מעל 500 שורות נחשבים גדולים
- מקבלים טיפול מיוחד בטעינה ותצוגה

### תגיות 🏷️

**איך לראות תגיות:**
- שלח `/tags` כדי לראות את כל התגיות שלך עם מספר הקבצים

**איך להוסיף תגיות:**
- בעת שמירת קובץ חדש, הוסף `#tag` אחרי שם הקובץ
- לדוגמה: `api.py #python #api #rest`

**חיפוש לפי תגית:**
- דרך **🔎 חפש קובץ**, השתמש ב-`tag:tagname`

### קבצים אחרונים

**איך:**
- קבצים שנוצרו או עודכנו לאחרונה מוצגים בראש הרשימות
- ניתן גם לסנן לפי תאריך דרך החיפוש

## 🔍 ניתוח ובדיקת קוד

### ניתוח קובץ 📊

**איך:**
1. פתח את הקובץ דרך **👁️ הצג קוד**
2. בתצוגת הקוד, לחץ על **📊 ניתוח**
3. הבוט יבצע ניתוח מעמיק

**מה מנותח:**
- **סטטיסטיקות:**
  - מספר שורות קוד
  - שורות הערות
  - שורות ריקות
  - מספר פונקציות ומחלקות
  
- **מדדי מורכבות:**
  - מורכבות ציקלומטית
  - עומק קינון
  - אורך פונקציות
  
- **פונקציות שנמצאו:**
  - רשימה של כל הפונקציות בקוד
  - מיקום (שורה) של כל פונקציה
  
- **המלצות לשיפור:**
  - הצעות לאופטימיזציה
  - שיפורי קוד
  - אזהרות אבטחה פוטנציאליות

**פקודה חלופית:**
- ניתן גם להשתמש ב-`/analyze filename.py`

### בדיקת תקינות ✅

**איך:**
- שלח `/validate filename.py`

**מה נבדק:**
- שגיאות תחביר
- ייבואים חסרים
- משתנים לא מוגדרים
- בעיות לוגיות בסיסיות
- תאימות לשפת התכנות

### היסטוריית גרסאות 📚

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **📚 היסטוריה**
3. כל הגרסאות יוצגו עם תאריכים
4. לחץ על גרסה כדי לצפות בה
5. לחץ על **שחזר** כדי להחזיר גרסה קודמת

## 🌐 שיתוף וייצוא

### שיתוף קובץ 🔗

**איך:**
1. פתח את תפריט הקובץ
2. לחץ על **🔗 שתף קוד**
3. תפריט שיתוף יופיע עם אפשרויות:
   - 🐙 **GitHub Gist** - שיתוף ב-GitHub Gist (דורש הגדרת טוקן GitHub)
   - 📋 **Pastebin** - שיתוף ב-Pastebin (דורש הגדרת API Key)
   - 📱 **קישור פנימי** - קישור זמני מאובטח (תוקף מוגבל)
4. בחר את שיטת השיתוף הרצויה
5. הקישור יישלח אליך

**מה קורה:**
- **GitHub Gist:** יוצר Gist ציבורי עם הקוד
- **Pastebin:** מעלה את הקוד ל-Pastebin
- **קישור פנימי:** יוצר קישור זמני שניתן לשתף

**פקודה חלופית:**
- ניתן גם להשתמש ב-`/share filename.py`

### יצירת ZIP 🗜️

**איך:**
1. לחץ על **🗜️ יצירת ZIP** בתפריט הראשי
2. בחר קבצים:
   - שלח שמות קבצים (אחד בכל שורה)
   - או לחץ על כפתורי קבצים מהרשימה
3. לחץ על **✅ סיום** כשסיימת
4. הבוט ייצור קובץ ZIP וישלח לך

**מה אפשר לכלול:**

**דוח סיכום כולל:**
- מספר קבצים שנבדקו

## 💡 טיפים ושימוש מתקדם

### ארגון יעיל של קבצים

1. **שימוש בתגיות:**
   - הוסף תגיות עקביות בעת שמירה (למשל: `api.py #python #api`)
   - השתמש בתגיות שמורות: `#python`, `#javascript`, `#api`, `#utils`
   - צור תגיות מותאמות אישית לפי הפרויקט

2. **שמות קבצים ברורים:**
   - השתמש בשמות תיאוריים בעת שמירה
   - ניתן לשנות שם אחר כך דרך **📝 שנה שם** בתפריט הקובץ
   - הוסף קידומות: `api_`, `utils_`, `test_`
   - הימנע משמות גנריים כמו `script.py`

3. **מועדפים:**
   - סמן קבצים חשובים כמועדפים דרך כפתור **⭐** בתצוגת הקובץ
   - צפה במועדפים דרך **⭐ מועדפים** בתפריט הקבצים
   - השתמש במועדפים לגישה מהירה לקבצים חשובים

### חיפוש יעיל

1. **דרך תפריט הקבצים:**
   - לחץ על **🔎 חפש קובץ** בתפריט הקבצים
   - השתמש בסינונים: `name:`, `lang:`, `tag:`
   - שילוב קריטריונים: `name:util lang:python`

2. **חיפוש בתוכן:**
   - החיפוש מחפש גם בתוכן הקוד, לא רק בשמות
   - פשוט כתוב מילת מפתח והבוט יחפש בכל הקבצים

3. **ארגון לפי ריפו:**
   - השתמש ב-**🗂 לפי ריפו** לארגון פרויקטים
   - קבצים שיובאו מ-GitHub מסודרים אוטומטית

### עבודה עם GitHub

1. **טוקן מאובטח:**
   - הגדר טוקן דרך **🔧 GitHub** > **🔐 הגדרת טוקן**
   - צור טוקן עם הרשאות מינימליות ב-GitHub
   - אם מוגדר `TOKEN_ENC_KEY`, הטוקן יישמר מוצפן
   - ניתן למחוק את הטוקן בכל עת דרך **🚪 התנתקות**

2. **ייבוא פרויקטים:**
   - השתמש ב-**📥 ייבוא ZIP מריפו** בתפריט הראשי
   - ייבוא שומר את מבנה התיקיות
   - כל קובץ נשמר בנפרד עם תגית `repo:owner/project-name`
   - ניתן לארגן לפי ריפו דרך **🗂 לפי ריפו**

3. **העלאת קבצים:**
   - דרך **🔧 GitHub** > **📤 העלה קובץ**
   - בחר ריפו ותיקייה
   - הקובץ יועלה ל-GitHub

4. **ניתוח פרויקטים:**
   - השתמש ב-**📊 ניתוח ריפו** או **✅ בדוק תקינות ריפו**
   - מקבל דוחות מפורטים על הפרויקט

### ניתוח קוד יעיל

1. **ניתוח קובץ יחיד:**
   - פתח את הקובץ דרך **👁️ הצג קוד**
   - לחץ על **📊 ניתוח** בתצוגת הקובץ
   - בדוק את ההמלצות לשיפור

2. **ניתוח Batch:**
   - השתמש ב-**⚡ עיבוד Batch** > **📊 ניתוח Batch**
   - בחר קטגוריה (כל הקבצים/לפי שפה/לפי ריפו)
   - עיין בדוחות הסיכום לקבלת תמונה כוללת

3. **בדיקת תקינות:**
   - בדיקה בודדת: פתח קובץ ולחץ על כפתור ניתוח
   - Batch validation: דרך **⚡ עיבוד Batch** > **✅ בדיקת תקינות Batch**
   - עוזר למצוא בעיות במהירות בפרויקטים גדולים

### שימוש בתזכורות

**איך:**
- שלח `/remind` ליצירת תזכורת
- שלח `/reminders` לראות את כל התזכורות

**טיפים:**
1. **תזכורות קצרות:**
   - השתמש בתזכורות לבדיקת קוד לאחר זמן
   - פורמטים נוחים: `בעוד שעה`, `מחר 10:00`

2. **תזכורות מהירות:**
   - פורמט מהיר: `/remind "לבדוק את הקוד" בעוד שעה`
   - חוסך זמן במקום הזרימה האינטראקטיבית

### עבודה עם קבצים גדולים

1. **מתי להשתמש:**
   - קבצים מעל 500KB מומלץ לשמור כקבצים גדולים
   - קבצים גדולים נשמרים בצורה מיוחדת

2. **ניהול:**
   - השתמש ב-`/large` לניהול קבצים גדולים
   - קבצים גדולים יכולים להשפיע על ביצועים

### דפוסי עבודה מומלצים

1. **שמירה תכופה:**
   - כל עריכה דרך **✏️ ערוך** יוצרת גרסה חדשה
   - הגרסאות הישנות נשמרות אוטומטית
   - השתמש ב-**📚 היסטוריה** לשחזור שינויים

2. **גיבויים:**
   - צור גיבויים קבועים דרך `/backup`
   - שמור גיבויים לפני שינויים גדולים
   - הוסף הערות לגיבויים כדי לזכור מה השתנה

3. **תיעוד:**
   - הוסף הערות דרך **📝 ערוך הערה** בתפריט הקובץ
   - השתמש בתגיות לארגון בעת שמירה
   - תגיות עוזרות למצוא קבצים מהר

4. **עבודה עם פרויקטים:**
   - ייבא פרויקטים דרך **📥 ייבוא ZIP מריפו**
   - השתמש ב-**🗂 לפי ריפו** לארגון
   - ניתוח Batch עוזר לבדוק את כל הפרויקט

## ❓ שאלות נפוצות

### איך מוסיפים הערות/תיאור לקובץ?
1. פתח את תפריט הקובץ (לחץ על שם הקובץ)
2. לחץ על **📝 ערוך הערה**
3. כתוב את התיאור/הערה
4. התיאור יתעדכן ויוצג בתפריט הקובץ

### האם ניתן לעבוד עם כמה פרויקטים במקביל?
כן! 
- ייבא פרויקטים דרך **📥 ייבוא ZIP מריפו**
- השתמש ב-**🗂 לפי ריפו** לארגון פרויקטים
- כל פרויקט מסומן אוטומטית עם תגית `repo:owner/project-name`
- ניתן גם להשתמש בתגיות מותאמות אישית

### איך מייצאים את כל הקבצים?
1. לחץ על **🗜️ יצירת ZIP** בתפריט הראשי
2. בחר את כל הקבצים (או רק מה שתרצה)
3. לחץ על **✅ סיום**
4. הבוט ישלח לך קובץ ZIP עם כל הקבצים

או דרך תפריט גיבוי:
1. שלח `/backup`
2. לחץ על **📤 ייצוא ZIP**

### האם הבוט שומר את הקוד שלי באופן מאובטח?
כן. 
- הקוד נשמר במסד נתונים מאובטח
- אם מוגדר `TOKEN_ENC_KEY`, טוקני GitHub נשמרים מוצפנים
- ניתן למחוק את הטוקן בכל עת דרך **🚪 התנתקות** בתפריט GitHub

### איך משתפים קוד עם אחרים?
1. פתח את תפריט הקובץ
2. לחץ על **🔗 שתף קוד**
3. בחר שיטת שיתוף:
   - 🐙 **GitHub Gist** - לשיתוף ציבורי
   - 📋 **Pastebin** - אלטרנטיבה
   - 📱 **קישור פנימי** - שיתוף פרטי עם תוקף מוגבל
4. הקישור יישלח אליך

### האם ניתן לערוך קוד ישירות מהבוט?
כן!
1. פתח את תפריט הקובץ (לחץ על שם הקובץ)
2. לחץ על **✏️ ערוך**
3. שלח את הקוד החדש (כולו)
4. הגרסה הישנה תישמר בהיסטוריה אוטומטית

### איך מוצאים קבצים ישנים?
1. לחץ על **📚 הצג את כל הקבצים שלי**
2. לחץ על **🔎 חפש קובץ**
3. כתוב מילת מפתח או שם קובץ
4. הבוט יחפש בכל הקבצים שלך

קבצים אחרונים מוצגים בדרך כלל בראש הרשימות.
