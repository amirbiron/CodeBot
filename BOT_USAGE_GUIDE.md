## 🚀 התחלה מהירה

### תחילת עבודה עם הבוט

1. **התחלה ראשונה:**
   ```
   /start
   ```
   - מציג תפריט ראשי עם כל האפשרויות
   - רושם אותך במערכת
   - מכין את סביבת העבודה שלך

2. **קבלת עזרה:**
   ```
   /help
   ```
   - מציג הסברים מפורטים על כל הפונקציות
   - כולל דוגמאות שימוש

3. **תפריט ראשי:**
   לאחר `/start`, תראה תפריט עם כפתורים:
   - ➕ **הוסף קוד חדש** - לשמירת קטע קוד חדש
   - 📚 **הצג את כל הקבצים שלי** - רשימת כל הקבצים שלך
   - ⭐ **מועדפים** - קבצים מסומנים כמועדפים
   - 📂 **קבצים גדולים** - ניהול קבצים גדולים
   - 🔧 **GitHub** - אינטגרציה עם GitHub
   - 📥 **ייבוא ZIP מריפו** - ייבוא פרויקטים מ-GitHub
   - 🗜️ **יצירת ZIP** - יצירת ארכיון מקבצים
   - 🗂 **לפי ריפו** - ארגון קבצים לפי ריפוזיטורי
   - ℹ️ **הסבר על הבוט** - עזרה נוספת

## 📝 פקודות בסיסיות

### `/start` - התחלת עבודה
מתחיל את הבוט ומציג תפריט ראשי.
```bash
/start
```

### `/help` - עזרה מפורטת
מציג מידע על כל הפקודות והפונקציות.
```bash
/help
```

### `/save` - שמירת קוד חדש
שומר קטע קוד חדש במערכת.

**שימוש:**
```
/save filename.py
/save script.js #javascript #web
/save utils.py #python #helpers #utilities
```

**תהליך שמירה:**
1. שולח את הפקודה עם שם הקובץ
2. הבוט יבקש ממך לשלוח את הקוד
3. שלח את הקוד בדיוק כפי שתרצה שיישמר
4. הבוט ישמור את הקוד עם:
   - שם הקובץ
   - שפת התכנות (זיהוי אוטומטי)
   - תגיות (אם ציינת)
   - תאריך ושעה

**תגיות:**
- הוספת תגיות עם `#` לפני שם הקובץ
- דוגמה: `/save api.py #python #api #rest`
- עוזר לארגן ולמצוא קבצים מהר יותר

**דוגמה מלאה:**
```
👤 /save calculator.py #python #math

🤖 מוכן לשמור את calculator.py
   🏷️ תגיות: python, math
   אנא שלח את קטע הקוד:

👤 def add(a, b):
      return a + b

   def multiply(a, b):
      return a * b

🤖 ✅ נשמר בהצלחה!
   📁 calculator.py
   🔤 שפה: python
   🏷️ תגיות: python, math
   📊 גודל: 89 תווים
```

- מספר התגיות

## 📁 ניהול קבצים

### `/show` - הצגת קובץ
מציג קובץ עם אפשרויות פעולה נוספות.

**שימוש:**
```
/show filename.py
/show script.js
```

**מה מוצג:**
- תוכן הקובץ עם הדגשת תחביר צבעונית
- מידע על הקובץ (שפה, תאריך, גודל)
- כפתורי פעולה:
  - ✏️ **עריכה** - עריכת הקובץ
  - ⭐ **מועדף** - הוספה/הסרה ממועדפים
  - 🌐 **שיתוף** - שיתוף הקובץ
  - 📥 **הורדה** - הורדת הקובץ
  - 📊 **ניתוח** - ניתוח מתקדם
  - 📋 **העתקה** - העתקת תוכן
  - 🔄 **גרסאות** - צפייה בגרסאות קודמות

**דוגמה:**
```
/show calculator.py

📄 calculator.py
🔤 Python | 📅 05/01/2025 14:30
📏 89 תווים

[קוד מוצג עם הדגשה צבעונית]

[כפתורי פעולה]
```

### `/edit` - עריכת קובץ
מאפשר לערוך קובץ קיים.

**שימוש:**
```
/edit filename.py
```

**תהליך עריכה:**
1. שולח `/edit filename.py`
2. הבוט יציג את הקוד הנוכחי
3. שלח את הקוד החדש (כולו)
4. הבוט ישמור את הגרסה החדשה
5. הגרסה הישנה תישמר בהיסטוריה

**הערה:** עריכה יוצרת גרסה חדשה. הגרסאות הישנות נשמרות וניתן לשחזר אותן.

### `/delete` - מחיקת קובץ
מוחק קובץ מהמערכת.

**שימוש:**
```
/delete filename.py
```

**אזהרה:** מחיקה היא בלתי הפיכה. ודא שאתה רוצה למחוק לפני השימוש.

### `/download` - הורדת קובץ
מוריד קובץ כקובץ נפרד.

**שימוש:**
```
/download filename.py
```

הבוט ישלח לך את הקובץ כ-Document שניתן להוריד.

### `/info` - מידע על קובץ
מציג מידע מפורט על קובץ.

**מה מוצג:**
- שם הקובץ
- שפת התכנות
- תאריך יצירה ועדכון אחרון
- גודל הקובץ
- תגיות
- מספר גרסאות
- מספר שורות קוד
- מידע נוסף

**שימוש:**
```
/info filename.py
```

## 🔧 אינטגרציית GitHub

### `/github` - תפריט GitHub
פותח תפריט לניהול אינטגרציה עם GitHub.

**שימוש:**
```
/github
```

**אפשרויות:**
- 🔐 **הגדרת טוקן** - הגדרת GitHub Personal Access Token
- 📂 **בחר ריפו** - בחירת ריפוזיטורי לסנכרון
- 📤 **העלה קובץ** - העלאת קובץ ל-GitHub
- 📥 **ייבוא ZIP** - ייבוא פרויקט שלם מ-GitHub
- 📊 **ניתוח ריפו** - ניתוח פרויקט GitHub
- 🔄 **סנכרון** - סנכרון קבצים עם GitHub
- 🚪 **התנתקות** - הסרת טוקן GitHub

### הגדרת GitHub Token

**לפני השימוש:**
1. עבור ל-[GitHub Settings > Developer settings > Personal access tokens](https://github.com/settings/tokens)
2. צור Personal Access Token חדש
3. בחר הרשאות: `gist`, `repo` (לסנכרון מלא)
4. העתק את הטוקן

**הגדרה בבוט:**
1. שלח `/github`
2. בחר "הגדרת טוקן"
3. שלח את הטוקן (הפורמט: `ghp_...` או `github_pat_...`)
4. הטוקן יישמר מוצפן (אם מוגדר `TOKEN_ENC_KEY`)

### העלאת קבצים ל-GitHub

1. שלח `/github`
2. בחר "העלה קובץ"
3. בחר ריפוזיטורי
4. בחר תיקייה (אופציונלי)
5. שלח את הקובץ או הקוד
6. הקובץ יועלה ל-GitHub

### ייבוא פרויקט מ-GitHub

1. שלח `/github`
2. בחר "ייבוא ZIP"
3. בחר ריפו
4. בחר branch (אופציונלי, ברירת מחדל: main)
5. הבוט יייבוא את כל הקבצים מהפרויקט

### `/github_logout` - התנתקות מ-GitHub
מסיר את הטוקן GitHub מהמערכת.

**שימוש:**
```
/github_logout
```

פותח תפריט לניהול גיבויים.

**שימוש:**
```
/backup
```

**אפשרויות:**
- 📦 **צור גיבוי** - יצירת גיבוי של כל הקבצים שלך

4. בחר קבצים לשחזור (או כל הקבצים)
5. הבוט ישחזר את הקבצים

## ⏰ תזכורות

### `/remind` - יצירת תזכורת
יוצר תזכורת שתשלח בזמן מוגדר.

**שימוש בסיסי:**
```
/remind
```
הבוט ידריך אותך בתהליך היצירה.

**שימוש מהיר:**
```
/remind "לבדוק את הקוד" tomorrow 10:00
/remind "לעדכן דוקומנטציה" בעוד שעה
/remind "לבדוק PR" בעוד 15 דקות
```

**פורמטים נתמכים:**
- `בעוד 15 דקות` / `בעוד שעה` / `בעוד יום`
- `מחר 10:00` / `מחר בבוקר`
- `2025-01-15 14:30` (תאריך ושעה מלא)
- `בעוד שבוע`

**תהליך יצירה (אינטראקטיבי):**
1. שלח `/remind`
2. כתוב את כותרת התזכורת
3. בחר מתי להזכיר (מתפריט או הזן ידנית)
4. הוסף תיאור (אופציונלי)
5. התזכורת נוצרה!

## ⚡ עיבוד Batch

### `/batch_analyze` - ניתוח Batch
מבצע ניתוח על מספר קבצים בבת אחת.

**שימוש:**
```
/batch_analyze all              # כל הקבצים
/batch_analyze python           # קבצי Python בלבד
/batch_analyze file1.py file2.js   # קבצים ספציפיים
```

**תהליך:**
1. הבוט מתחיל את הניתוח ברקע
2. תקבל הודעת סטטוס עם Job ID
3. תוכל לבדוק סטטוס בכל עת
4. בסיום תקבל דוח מפורט

**דוח סיכום כולל:**
- מספר קבצים שנבדקו
- בעיות שנמצאו (אם יש)
- סטטיסטיקות כוללות
- המלצות לשיפור

**דוגמה:**
```
/batch_analyze all

⚡ ניתוח Batch התחיל!

📁 מנתח 42 קבצים
🆔 Job ID: job_abc123

⏱️ זמן משוער: 84 שניות

[📊 בדוק סטטוס] [❌ בטל]

[לאחר סיום]

📊 ניתוח הושלם!
✅ 38 קבצים תקינים
⚠️ 4 קבצים עם אזהרות
❌ 0 קבצים עם שגיאות

[כפתור: צפה בדוח המלא]
```

### `/batch_validate` - בדיקת תקינות Batch
בודק תקינות תחביר של מספר קבצים בבת אחת.

**שימוש:**
```
/batch_validate all
/batch_validate javascript
/batch_validate file1.py file2.py
```

**מה נבדק:**
- שגיאות תחביר בכל קבצים
- ייבואים חסרים
- בעיות בסיסיות

### `/job_status` - בדיקת סטטוס עבודה
בודק את הסטטוס של עבודת Batch.

**שימוש:**
```
/job_status job_abc123
```

**מה מוצג:**
- סטטוס העבודה (בביצוע/הושלם/נכשל)
- התקדמות (אחוז)
- זמן משוער לסיום
- תוצאות חלקיות (אם יש)

## 🧹 ניקוי ושמירת קוד

### ניקוי אוטומטי
הבוט מנקה את הקוד אוטומטית לפני שמירה כדי למנוע בעיות:

**מה מנוקה:**
- תווי BOM בתחילת הקובץ
- המרת שורות (CRLF/CR → LF)
- החלפת תווי רוחב-אפס ותווי בקרה
- הסרת רווחים בסוף שורות

**הערה:** ניקוי זה לא משנה את הלוגיקה של הקוד, רק מנקה תווים נסתרים שעלולים לגרום לבעיות.

**הגדרה:**
ניתן לכבות את הניקוי האוטומטי דרך משתנה סביבה:
```env
NORMALIZE_CODE_ON_SAVE=false
```

## 💡 טיפים ושימוש מתקדם

### ארגון יעיל של קבצים

1. **שימוש בתגיות:**
   - הוסף תגיות עקביות לכל קובץ
   - השתמש בתגיות שמורות: `#python`, `#javascript`, `#api`, `#utils`
   - צור תגיות מותאמות אישית לפי הפרויקט

2. **שמות קבצים ברורים:**
   - השתמש בשמות תיאוריים
   - הוסף קידומות: `api_`, `utils_`, `test_`
   - הימנע משמות גנריים כמו `script.py`

3. **מועדפים:**
   - סמן קבצים חשובים כמועדפים
   - השתמש במועדפים לגישה מהירה

### חיפוש יעיל

1. **שילוב קריטריונים:**
   ```
   /search #api python
   ```
   מחפש קבצי Python עם תגית api.

2. **חיפוש בתוכן:**
   ```
   /search "async function"
   ```
   מחפש בתוכן הקוד, לא רק בשמות.

3. **שימוש ב-recent:**
   השתמש ב-`/recent` כדי למצוא קבצים שעובדת עליהם לאחרונה.

### עבודה עם GitHub

1. **טוקן מאובטח:**
   - צור טוקן עם הרשאות מינימליות
   - אם מוגדר `TOKEN_ENC_KEY`, הטוקן יישמר מוצפן
   - ניתן למחוק את הטוקן בכל עת דרך `/github_logout`

2. **סנכרון אוטומטי:**
   - לאחר הגדרת טוקן וריפו, ניתן לסנכרן קבצים אוטומטית
   - השתמש ב-`/github` לניהול הסנכרון

3. **ייבוא פרויקטים:**
   - ייבוא ZIP מ-GitHub שומר את מבנה התיקיות
   - כל קובץ נשמר בנפרד עם תגית הריפו המקורי

### ניתוח קוד יעיל

1. **ניתוח לפני שמירה:**
   - השתמש ב-`/analyze` לפני שמירת קוד חשוב
   - בדוק את ההמלצות לשיפור

2. **Batch Analysis:**
   - השתמש ב-`/batch_analyze` לניתוח פרויקטים שלמים
   - עיין בדוחות הסיכום לקבלת תמונה כוללת

3. **בדיקת תקינות:**
   - השתמש ב-`/validate` לפני שיתוף קוד
   - Batch validation עוזר למצוא בעיות במהירות

### שימוש בתזכורות

1. **תזכורות קצרות:**
   - השתמש בתזכורות לבדיקת קוד לאחר זמן
   - תזכורות עוזרות לזכור משימות חשובות

2. **תזכורות תקופתיות:**
   - צור תזכורות לבדיקות קבועות
   - השתמש בתזכורות לעדכונים תקופתיים

### עבודה עם קבצים גדולים

1. **מתי להשתמש:**
   - קבצים מעל 500KB מומלץ לשמור כקבצים גדולים
   - קבצים גדולים נשמרים בצורה מיוחדת

2. **ניהול:**
   - השתמש ב-`/large` לניהול קבצים גדולים
   - קבצים גדולים יכולים להשפיע על ביצועים

### דפוסי עבודה מומלצים

1. **שמירה תכופה:**
   - שמור גרסאות מרובות של אותו קובץ
   - השתמש בגרסאות לשחזור שינויים

2. **גיבויים:**
   - צור גיבויים קבועים עם `/backup`
   - שמור גיבויים לפני שינויים גדולים

3. **תיעוד:**
   - הוסף הערות בכותרת הקובץ
   - השתמש בתגיות לארגון

## ❓ שאלות נפוצות

### איך מוסיפים הערות/הערה לקובץ?
בעת שמירה, הבוט יכול לשאול על הערות. ניתן גם לערוך קובץ ולהוסיף הערות בתחילתו.

### האם ניתן לעבוד עם כמה פרויקטים במקביל?
כן! השתמש בתגיות כדי לארגן קבצים לפי פרויקט. לדוגמה: `#project1`, `#project2`.

### איך מייצאים את כל הקבצים?
השתמש ב-`/backup` ובחר "ייצוא ZIP" ליצירת קובץ ZIP עם כל הקבצים.

### האם הבוט שומר את הקוד שלי באופן מאובטח?
כן. הקוד נשמר במסד נתונים מאובטח. אם מוגדר `TOKEN_ENC_KEY`, טוקנים נשמרים מוצפנים.

### איך משתפים קוד עם אחרים?
השתמש ב-`/share` לבחירת שיטת שיתוף:
- GitHub Gist - לשיתוף ציבורי
- Pastebin - אלטרנטיבה
- קישור פנימי - שיתוף פרטי עם תוקף מוגבל

### האם ניתן לערוך קוד ישירות מהבוט?
כן, השתמש ב-`/edit filename.py` וצור את הקוד החדש. הגרסה הישנה תישמר בהיסטוריה.

### איך מוצאים קבצים ישנים?
השתמש ב-`/search` עם מילות מפתח או ב-`/recent` עם טווח תאריכים גדול יותר.
