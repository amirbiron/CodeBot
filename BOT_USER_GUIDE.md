## 🚀 התחלה מהירה

### 1. הפעלת הבוט
פשוט שלח את הפקודה הבאה בצ'אט עם הבוט:
```
/start
```

הבוט יציג לך תפריט ראשי עם כל האפשרויות הזמינות.

### 2. שמירת קוד ראשון
```
/save hello.py

# לאחר מכן שלח את הקוד:
def hello_world():
    print("Hello, World!")
```

### 3. קבלת עזרה
```
/help
```

## 📁 ניהול קבצים

### `/save <שם_קובץ> [#תגיות]`
**תיאור:** שומר קוד חדש במערכת

**דוגמאות:**
```
# שמירה פשוטה
/save script.py

# שמירה עם תגיות
/save api_client.py #python #api #rest

# שמירה עם תיאור
/save utils.py
תיאור: פונקציות עזר כלליות למערכת
```

**תכונות:**
- זיהוי שפת תכנות אוטומטי
- תמיכה בתגיות מרובות
- תמיכה בתיאור מפורט
- ניקוי קוד אוטומטי (הסרת BOM, NBSP, ZWSP וכו')
- הדגשת תחביר צבעונית

---

### `/list [מס_עמוד]`
**תיאור:** מציג רשימה של כל הקבצים שלך

**דוגמאות:**
```
# רשימה פשוטה
/list

# עמוד ספציפי
/list 2
```

**תכונות:**
- עימוד (10 קבצים בעמוד)
- כפתורי ניווט "הבא/הקודם"
- מידע על שפה, תאריך ותגיות
- כפתור מהיר לצפייה בקובץ

---

### `/show <שם_קובץ>`
**תיאור:** מציג קובץ עם אפשרויות פעולה

**דוגמה:**
```
/show script.py
```

**תצוגה:**
- הקוד עצמו עם הדגשת תחביר
- מידע על הקובץ (גודל, שפה, תאריך)
- כפתורים:
  - 🎨 הדגשה צבעונית
  - ✏️ עריכה
  - 🗑️ מחיקה
  - 📊 ניתוח
  - 🌐 שיתוף
  - 📥 הורדה
  - 📋 העתקה

---

### `/edit <שם_קובץ>`
**תיאור:** עורך קובץ קיים

**דוגמה:**
```
/edit script.py

```

**תכונות:**
- שומר גרסה קודמת אוטומטית
- מציג diff בין הגרסאות
- אפשרות לבטל שינויים

---

### `/delete <שם_קובץ>`
**תיאור:** מוחק קובץ (עם אישור)

**דוגמה:**
```
/delete old_script.py
```

**תכונות:**
- מסך אישור למניעת מחיקה בטעות
- אפשרות שחזור מגיבויים

---

### `/rename <שם_ישן> <שם_חדש>`

**תכונות:**
- שומר גרסה נוכחית לפני שחזור

### `/diff <שם_קובץ> <גרסה1> <גרסה2>`


**תצוגה:**
- שורות שנוספו (ירוק)
- שורות שהוסרו (אדום)
- שורות ששונו (כתום)

## 🐙 אינטגרציה עם GitHub

הבוט כולל אינטגרציה מלאה עם GitHub לניהול repositories מתוך Telegram.

### הגדרה ראשונית

**1. הגדרת טוקן GitHub**
```
/github → 🔑 הגדר טוקן GitHub
```

**צעדים:**
1. עבור ל-[GitHub Settings > Tokens](https://github.com/settings/tokens)
2. צור Personal Access Token חדש
3. בחר הרשאות: `repo`, `gist`
4. העתק את הטוקן
5. שלח לבוט

**אבטחה:**
- הטוקן מוצפן במסד הנתונים
- אפשרות למחיקה בכל עת
- לא נשלח לשירותים חיצוניים

---

**2. בחירת Repository**
```
/github → 📁 בחר ריפו
```

הבוט יציג רשימת הריפוזיטוריז שלך (עם עימוד).

---

### דפדפן קבצים GitHub

**📥 הורדת קבצים**
```
/github → 📥 הורד קובץ מריפו
```

**תכונות:**
- ניווט בתיקיות (Breadcrumbs)
- עימוד (10 פריטים בעמוד)
- הורדת קובץ בודד
- הורדת תיקייה שלמה כ-ZIP

---

**🗑️ מחיקת קבצים**
```
/github → 🗑️ מחק קובץ
```

**מצבי מחיקה:**

**מחיקה ישירה**
- מחיקה מיידית מה-repository
- דורש הרשאות כתיבה

**מחיקה בטוחה (Safe Delete)**
- יוצר Pull Request עם המחיקות
- אפשרות לסקירה לפני merge
- מומלץ למניעת טעויות

---

**✅ בחירה מרובה**
```
/github → ✅ בחר מרובים
```

**פעולות:**
- 📦 הורדת קבצים נבחרים כ-ZIP
- 🗑️ מחיקה מרובה
- 🔗 שיתוף קישורים לקבצים

---

### ניהול Pull Requests

**🔀 Pull Requests**
```
/github → 🔀 Pull Requests
```

```
1. בחר branch מקור (head)

3. PR נפתח ב-GitHub
```

**🔀 מיזוג PR פתוח**
```
1. רשימת PRים פתוחים (עם עימוד)
2. בחירת PR למיזוג
3. אישור
4. Merge מתבצע אוטומטית
```

### התראות חכמות

**🔔 התראות חכמות**
```
/github → 🔔 התראות חכמות
```

**הגדרות:**
- הפעלה/כיבוי התראות

**תכונות:**
- התראות על PRs חדשים
- התראות על Issues חדשים
- עדכונים על PRs/Issues קיימים
- קישורים ישירים לאירועים

### שאילתות Inline (Inline Queries)

הבוט תומך בשאילתות inline מכל צ'אט:

**דוגמאות:**
```
# הורדת תיקייה כ-ZIP
@שם_הבוט zip src/components

# הורדת קובץ ספציפי
@שם_הבוט file README.md
@שם_הבוט file src/utils/helper.py

# חיפוש בתיקייה
@שם_הבוט src/
```

**הערה:** יש להפעיל Inline Mode אצל [@BotFather](https://t.me/BotFather) לפני שימוש.


**מה זה עושה:**
- יוצר tag קל-משקל על HEAD
- שם בפורמט: `checkpoint-YYYYMMDD-HHMMSS`
- אם נכשל, יוצר branch במקום

**שחזור מנקודת שמירה:**
```bash
# משיכת tags
git fetch --tags

# מעבר ל-tag
git checkout tags/checkpoint-20250815-143000

# יצירת branch מ-tag
git checkout -b restore-from-checkpoint
```

## 💡 טיפים ושימושים מתקדמים

### 1. ארגון קבצים עם תגיות
השתמש בתגיות עקביות לארגון טוב יותר:
```
#frontend #react #component
#backend #api #python
#utils #helper #common
#test #unittest #pytest
```

### 2. שימוש בחיפוש מתקדם
```
# מצא כל קבצי Python עם API
/search #python #api

# מצא קבצים שנוצרו החודש
/search created:month

# חיפוש מדויק
/search "def main():"
```

### 3. ניהול גרסאות יעיל
- שמור גרסאות עם תיאורים ברורים
- השתמש ב-diff לפני שחזור
- צור גיבוי לפני שינויים גדולים

### 4. אוטומציה עם Batch
```
# ניתוח כל קבצי Python
/batch analyze *.py

# תיוג אוטומטי לפי תיקייה
/batch tag src/frontend/* #frontend
/batch tag src/backend/* #backend
```

### 5. אינטגרציה עם GitHub
- שמור נקודות שמירה לפני שינויים גדולים
- השתמש במחיקה בטוחה (Safe Delete) למניעת טעויות
- הפעל התראות חכמות למעקב אחר הפרויקט

### 6. גיבויים אוטומטיים
- הפעל גיבוי אוטומטי שבועי
- שמור גיבויים חשובים במקומות נוספים
- בדוק גיבויים מדי פעם ווודא שהם תקינים

### 7. שיתוף יעיל
- השתמש ב-GitHub Gist לשיתוף ציבורי
- השתמש בקישורים פנימיים לשיתוף זמני
- הוסף תיאורים מפורטים לפני שיתוף

### 8. ניתוח קוד שוטף
- הרץ ניתוח על קבצים חדשים
- עקוב אחר מדדי מורכבות
- תקן בעיות מיד כשהן מזוהות
