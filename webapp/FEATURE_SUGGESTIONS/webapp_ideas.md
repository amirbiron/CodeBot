## 🔥 עדיפות גבוהה – שיפורים מיידיים

### 1. 🎯 Command Palette – לוח פקודות מהיר

**מה זה:**  
חלון קופץ (modal) שנפתח עם `Ctrl/Cmd+K` ומאפשר גישה מהירה לכל פעולה באפליקציה.

**איך זה עובד:**
- הקלדת חיפוש חופשי: "פתח קובץ", "עבור להגדרות", "חפש בתוכן"
- קיצורי דרך מוצגים ליד כל פקודה
- היסטוריית פקודות אחרונות
- פקודות מותאמות לקונטקסט (תלוי בעמוד הנוכחי)
- תמיכה ב-fuzzy search

**דוגמת פקודות:**
```
> קובץ חדש          [Ctrl+N]
> חפש בכל הקבצים   [Ctrl+Shift+F]
> עבור להגדרות     [Ctrl+,]
> החלף ערכת נושא   [Ctrl+Shift+T]
> יצוא לPDF         
> סימניות          [Ctrl+B]
```

**למה זה חשוב:**
- מהירות גישה לכל פעולה
- חוויית "power user"
- מפחית שימוש בעכבר
- מוכר ממערכות כמו VS Code, Notion, Slack

**מורכבות:** נמוכה-בינונית | **ROI:** גבוה מאוד | **זמן:** 3-5 ימים

### 2. 📊 סרגל מצב חכם (Smart Status Bar)

**מה זה:**  
סרגל תחתון קבוע בעמודי צפייה/עריכה עם מידע שימושי על הקובץ.

**מה מציג:**
- שורות / תווים / מילים
- שפת תכנות + אייקון
- תאריך עדכון אחרון
- גודל קובץ
- מצב שמירה (נשמר / שינויים ממתינים)
- Encoding (UTF-8)
- מיקום סמן (שורה:עמודה)
- זמן קריאה משוער (ל-Markdown)

**עיצוב:**
```
┌────────────────────────────────────────────────────────────────┐
│ 📄 142 שורות │ 🐍 Python │ 2.3KB │ עודכן לפני 3 שעות │ UTF-8 │
└────────────────────────────────────────────────────────────────┘
```

**למה זה חשוב:**
- מידע מיידי ללא צורך בגלילה
- מרגיש מקצועי כמו IDE
- שימושי במיוחד לקבצים גדולים

**מורכבות:** נמוכה | **ROI:** גבוה | **זמן:** 2-3 ימים

### 3. 🎨 מצב קריאה (Reading Mode)

**מה זה:**  
מצב תצוגה מותאם לקריאה ארוכה של קבצי Markdown ותיעוד.

**מה משתנה:**
- רוחב טקסט אופטימלי (60-70 תווים לשורה)
- גופן serif או san-serif גדול יותר
- ריווח שורות מוגדל (line-height: 1.8)
- הסתרת ה-sidebar והניווט
- מצב מסך מלא (F11)
- רקע רך לעיניים (גוון סֵפיה אופציונלי)
- מעבר קל עם `Ctrl+Shift+R`

**אופציות נוספות:**
- "מצב לילה" עם ניגודיות נמוכה
- גודל גופן מותאם (סליידר)
- הצגת זמן קריאה משוער

**למה זה חשוב:**
- קריאה נוחה של README ותיעוד ארוך
- הפחתת עומס ויזואלי
- חוויית קריאה כמו Medium או Notion

**מורכבות:** נמוכה | **ROI:** גבוה | **זמן:** 2-3 ימים

### 4. 📌 הצמדת קבצים (Pinned Files)

**מה זה:**  
אפשרות להצמיד קבצים חשובים לראש הרשימה או לסרגל צד.

**איך זה עובד:**
- כפתור "הצמד" ליד כל קובץ (סיכה 📌)
- קבצים מוצמדים מופיעים תמיד בראש `/files`
- סרגל צד קטן עם קבצים מוצמדים (אופציונלי)
- גרירה לשינוי סדר
- מגבלה: עד 10 קבצים מוצמדים

**שימושים:**
- קבצי קונפיג חשובים (.env, docker-compose)
- README של פרויקט
- קובץ הערות יומי
- Cheatsheet שחוזרים אליו

**למה זה חשוב:**
- גישה מהירה לקבצים הכי שימושיים
- לא מתבלבלים בין עשרות קבצים
- פשוט ויעיל

**מורכבות:** נמוכה | **ROI:** גבוה מאוד | **זמן:** 2-3 ימים

### 5. ⏱️ זמן קריאה משוער

**מה זה:**  
הצגת הערכת זמן קריאה לקבצי Markdown ותיעוד.

**איפה מוצג:**
- בכרטיס הקובץ בדף `/files`
- בכותרת עמוד תצוגת Markdown
- ב-Status Bar

**חישוב:**
```python
# ממוצע קריאה: 200-250 מילים לדקה (עברית/אנגלית)
words = len(content.split())
reading_time = max(1, round(words / 220))
# תוצאה: "⏱️ 3 דק' קריאה"
```

**למה זה חשוב:**
- מנהל ציפיות לפני כניסה לקובץ
- עוזר לתעדף מה לקרוא
- תחושת מקצועיות

**מורכבות:** נמוכה מאוד | **ROI:** בינוני-גבוה | **זמן:** יום אחד

**למה זה חשוב:**

- שימושי במיוחד לתיעוד ארוך

### 7. 📋 היסטוריית לוח העתקה (Clipboard History)

**מה זה:**  
שמירת קטעי קוד שהועתקו לאחרונה וגישה מהירה אליהם.

**איך זה עובד:**
- בכל העתקה (Ctrl+C או כפתור copy): שמירה בהיסטוריה
- פאנל צד או popup עם 10 הפריטים האחרונים
- קיצור: `Ctrl+Shift+V` לפתיחת ההיסטוריה
- לחיצה על פריט → הדבקה
- אחסון ב-localStorage (נשמר בין סשנים)

**תצוגה:**
```
┌─ היסטוריית העתקות ──────────────┐
│ 1. const x = 42;                │
│ 2. function hello() { ...       │
│ 3. SELECT * FROM users WHERE... │
│ 4. npm install --save-dev       │
└─────────────────────────────────┘
```

**למה זה חשוב:**
- עבודה עם מספר קטעי קוד במקביל
- לא צריך לחזור לקובץ המקור
- פרודוקטיביות משופרת

**מורכבות:** בינונית | **ROI:** בינוני | **זמן:** 3-4 ימים

### 8. 🗺️ מיני-מפה (Minimap) לקוד

**מה זה:**  
תצוגה מוקטנת של כל הקובץ בצד ימין/שמאל של העורך.

**איך זה עובד:**
- מציג את כל הקוד בגודל זעיר
- מדגיש את האזור הנראה כעת
- לחיצה על המיניmap → קפיצה למיקום
- highlight לשורות עם שגיאות/אזהרות
- סימון אזורי diff

**עיצוב:**
```
┌──────────────────────┬───┐
│                      │███│ ← אזור נוכחי
│   תוכן הקובץ         │   │
│                      │   │
│                      │ ▓ │ ← שגיאה
│                      │   │
└──────────────────────┴───┘
```

**למה זה חשוב:**
- ניווט מהיר בקבצים ארוכים
- תמונת מצב כללית של הקובץ
- מוכר מ-VS Code

**מורכבות:** בינונית-גבוהה | **ROI:** בינוני | **זמן:** 1-2 שבועות

### 9. 🔗 קישורים חכמים בקוד

**מה זה:**  
זיהוי אוטומטי של URLs, נתיבי קבצים, ומזהים בקוד והפיכתם לקישורים לחיצים.

**מה מזוהה:**
- URLs: `https://example.com` → קישור חיצוני
- נתיבים: `./config/settings.py` → קישור לקובץ במערכת (אם קיים)
- Issues: `#123` → קישור ל-GitHub Issue
- Stack Overflow: `stackoverflow.com/...` → פתיחה בטאב חדש
- npm packages: `"lodash": "^4.17"` → קישור ל-npmjs.com

**עיצוב:**
- קו תחתון מקווקו על hover
- tooltip עם preview (אופציונלי)
- `Ctrl+Click` לפתיחה

**למה זה חשוב:**
- ניווט מהיר בין קבצים קשורים
- גישה לתיעוד חיצוני
- קוד הופך לאינטראקטיבי

**מורכבות:** בינונית | **ROI:** בינוני | **זמן:** 4-5 ימים

### 10. 📑 תצוגת טאבים לקבצים פתוחים

**מה זה:**  
סרגל טאבים עליון המציג את הקבצים שנפתחו בסשן הנוכחי.

**איך זה עובד:**
- כל קובץ שנפתח מתווסף כטאב
- סגירה עם X או Middle-click
- גרירה לשינוי סדר
- טאב פעיל מודגש
- מגבלה: עד 8-10 טאבים (scroll אופקי מעבר לכך)
- שמירה ב-localStorage

**עיצוב:**
```
┌──────┬──────┬──────┬──────┬──────────────────────┐
│config│README│ app  │utils │                      │
│.yml  │.md ● │.py   │.js   │                      │
└──────┴──────┴──────┴──────┴──────────────────────┘
         ↑ שינויים לא שמורים
```

**למה זה חשוב:**
- מעבר מהיר בין קבצים קשורים
- חוויית IDE מוכרת
- עבודה יעילה יותר

**מורכבות:** בינונית | **ROI:** גבוה | **זמן:** 1 שבוע

### 11. ⚡ קיצורי מקלדת מותאמים אישית

**מה זה:**  
עמוד הגדרות המאפשר למשתמש להגדיר קיצורי מקלדת משלו.

**מה ניתן להגדיר:**
- כל פעולה באפליקציה: שמירה, חיפוש, ניווט
- תבניות טקסט: `;;today` → תאריך היום
- macros פשוטים: רצף פעולות

**ממשק:**
```
┌─ הגדרת קיצורים ────────────────────────────────┐
│ פעולה           │ קיצור נוכחי  │ שנה           │
│─────────────────┼──────────────┼───────────────│
│ שמירה           │ Ctrl+S       │ [הקלד קיצור]  │
│ חיפוש גלובלי    │ Ctrl+Shift+F │ [הקלד קיצור]  │
│ Command Palette │ Ctrl+K       │ [הקלד קיצור]  │
└─────────────────────────────────────────────────┘
```

**למה זה חשוב:**
- התאמה אישית לכל משתמש
- יעילות מקסימלית
- גמישות

**מורכבות:** בינונית | **ROI:** בינוני | **זמן:** 1 שבוע

### 13. 📊 דף סטטיסטיקות אישיות

### 14. 🌐 תמיכה בקבצי `.env` עם הצפנה

**מה זה:**  
תצוגה מיוחדת לקבצי environment variables עם אפשרות להסתיר ערכים רגישים.

**יכולות:**
- זיהוי אוטומטי של קבצי `.env`
- כפתור "הסתר ערכים" → מציג `API_KEY=********`

- אזהרה אם יש ערך חשוף בקובץ משותף
- highlighting לסינטקס env

**עיצוב:**
```
DATABASE_URL=postgres://... [👁️ הצג]
API_KEY=********************  [👁️ הצג]
DEBUG=true
```

**למה זה נחמד:**
- אבטחה משופרת
- בטוח להציג מסך
- שימושי לעבודה עם קונפיגים

**מורכבות:** נמוכה-בינונית | **ROI:** בינוני | **זמן:** 3-4 ימים

### 15. 🖼️ תצוגה מקדימה לתמונות בקוד

**מה זה:**

**איך זה עובד:**

**למה זה נחמד:**
- הבנה מהירה של assets בקוד
- לא צריך לצאת לחפש את התמונה
- שימושי לפרויקטי frontend

**מורכבות:** בינונית | **ROI:** נמוך | **זמן:** 4-5 ימים

### 17. 🔍 חיפוש והחלפה מתקדם

**מה זה:**  
מודאל חיפוש והחלפה עם יכולות מתקדמות.

**יכולות:**
- חיפוש רגיל / Regex / Case sensitive
- החלפה: בודד / הכל
- תצוגה מקדימה של שינויים (diff)
- היסטוריית חיפושים
- חיפוש בבחירה בלבד
- ספירת תוצאות בזמן אמת

**ממשק:**
```
┌─ חפש והחלף ─────────────────────────────────────┐
```

### ארכיטקטורה מומלצת

- שמירה ב-localStorage לפני שרת
- Debounce לאירועי input
