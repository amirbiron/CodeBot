# 💡 הצעות לשיפור WebApp - CodeBot (נובמבר 2025)

מסמך זה מרכז רעיונות לשיפור חוויית המשתמש והפרודוקטיביות בממשק ה-Web של הבוט.
ההצעות מבוססות על סריקת המצב הקיים (`webapp/`), בקשות משתמשים וסטנדרטים של עורכי קוד מודרניים.

> **מטרה:** להפוך את ה-WebApp מכלי צפייה בסיסי לסביבת עבודה עשירה, נוחה ויעילה.

---

## 🚀 פיצ'רים מובילים (Top Priority)

### 1. 🔖 Smart Bookmarks & Annotations (סימניות חכמות עם הערות)
**הצורך:** כרגע קיימת מערכת סימניות בסיסית, אך חסרה היכולת לסמן שורות ספציפיות ולהוסיף הקשר.
**ההצעה:**
*   **סימון שורה מדויק:** לחיצה על ה-Gutter (מספר השורה) בעורך תוסיף סימניה.
*   **הערות צפות (Annotations):** אפשרות להוסיף הערת טקסט לסימניה ("לבדוק למה זה קורס ב-Prod", "ריפקטורינג נדרש כאן").
*   **תצוגה צדדית:** פאנל "Annotations" המרכז את כל ההערות בקובץ הנוכחי לניווט מהיר.
*   **שמירה ב-DB:** הסימניות נשמרות פר-משתמש ופר-גרסת קובץ.

### 2. 🪞 MirrorCode Editor (עורך מראה)
**הצורך:** עריכת קוד דורשת לעיתים קרובות תצוגה מקבילה (קוד מול תוצאה, או קוד מול קוד אחר).
**ההצעה:** הוספת מצב עריכה חדש לצד העורך הקלאסי.
*   **מסך מפוצל (Split Pane):** צד אחד עורך (CodeMirror), צד שני תצוגה חיה (Preview) או קובץ אחר.
*   **Live Sync:** שינוי בקוד מעדכן את התצוגה בזמן אמת (ללא כפתור "רענן").
*   **שימושים:**
    *   עריכת HTML/CSS וראיית התוצאה מיידית.
    *   עריכת Python וראיית ה-Output (אם יחובר ל-Sandbox בעתיד).
    *   השוואת גרסאות (Diff View) בזמן אמת.

### 3. 📝 Markdown Live Experience (חוויית מרקדאון מתקדמת)
**הצורך:** קבצי `.md` הם קריטיים לתיעוד, אך התצוגה הנוכחית היא סטטית או דורשת מעבר בין מצבים.
**ההצעה:**
*   **תפריט צף (Floating Action Menu):** כפתור צף (`+`) בפינת המסך בטלפון/דסקטופ המאפשר פעולות מהירות (הוספת טבלה, הוספת תמונה, יצירת ToDo List).
*   **Instant Preview:** מעבר מיידי בין מצב "קוד" (Raw) למצב "תצוגה" (Rendered) בלחיצת כפתור או בטאב.
*   **Auto-TOC:** יצירת תוכן עניינים (Table of Contents) צף אוטומטית בצד המסך על בסיס הכותרות בקובץ.

### 4. 🌐 HTML Browser View (תצוגת דפדפן לקבצי HTML)
**הצורך:** צפייה בקבצי HTML כקוד בלבד אינה מספקת להבנת התוצר הסופי.
**ההצעה:**
*   **כפתור "Open in Browser View":** מרנדר את ה-HTML בתוך iframe מבודד (Sandboxed) בתוך ה-WebApp.
*   **Mobile Simulator:** מסגרת המדמה איך הדף נראה במובייל/טאבלט/דסקטופ.
*   **Inject CSS/JS:** אפשרות להזריק CSS חיצוני זמני לבדיקת עיצובים.

## 🛠️ שיפורי פרודוקטיביות ו-UI

### 5. 📂 Tabbed Interface (ממשק כרטיסיות)
**הצורך:** עבודה על מספר קבצים במקביל כיום דורשת מעבר עמודים או שימוש בכפתורי "חזור" של הדפדפן.
**ההצעה:**
*   **סרגל טאבים עליון:** כל קובץ שנפתח מתווסף לטאב עליון (כמו ב-VSCode/Chrome).
*   **שמירת Session:** חזרה ל-WebApp תפתח את הטאבים האחרונים שהיו פתוחים.
*   **Drag & Drop:** סידור טאבים בגרירה.
*   **אינדיקטור שינויים:** נקודה קטנה על הטאב אם יש שינויים שלא נשמרו.

### 6. 🔍 Global Search & Command Palette (חיפוש ופקודות)
**הצורך:** התמצאות מהירה בפרויקט עם קבצים רבים.
**ההצעה:**
*   **Command Palette (`Ctrl+Shift+P`):** חלונית חיפוש מרכזית המאפשרת:
    *   פתיחת קבצים (Fuzzy Search).
    *   הפעלת פקודות (למשל: "Toggle Dark Mode", "Format Code").
    *   ניווט לסימניות.
*   **Global Search:** חיפוש טקסט בתוך *תוכן* כל הקבצים בפרויקט (לא רק בשמות), עם תצוגת תוצאות מרוכזת.

### 7. 🧩 Code Folding & Tabs (קיפול וארגון קוד)
**הצורך:** קבצים ארוכים קשים להתמצאות.
**ההצעה:**
*   **Code Folding:** חיצים קטנים בצידי הקוד לקיפול פונקציות, מחלקות ובלוקים ארוכים.
*   **Explanation Tabs (כרטיסיות הסבר):** עבור קבצי לימוד או דוגמאות - אפשרות להוסיף "טאב הסבר" מעל קטע קוד, המכיל טקסט עשיר שמסביר את הלוגיקה (מעולה ל-Snippet Library).

### 8. ⭐ Quick Access & Favorites (גישה מהירה)
**הצורך:** גישה מיידית לקבצים בשימוש תכוף.
**ההצעה:**
*   **סרגל צד "מועדפים":** רשימה קבועה בצד המסך (ניתנת לקיפול).
*   **"נפתחו לאחרונה":** רשימה דינמית של 5-10 הקבצים האחרונים.
*   **Pinning:** אפשרות "לנעוץ" קובץ לראש הרשימה.

## 📱 חוויית מובייל (Mobile First)

### 9. ⌨️ Mobile Coding Toolbar
**ההצעה:** סרגל מקלדת וירטואלי מעל המקלדת הרגילה במובייל, המכיל תווים נפוצים בקוד שקשה להגיע אליהם (`{`, `}`, `[`, `]`, `=>`, `;`, `|`).

### 10. 👆 Swipe Actions
**ההצעה:**
*   החלקת קובץ ברשימה ימינה -> **שיתוף/שלח**.
*   החלקת קובץ ברשימה שמאלה -> **מחיקה/ארכיון**.

## ⚙️ הצעות טכניות ליישום

1.  **CodeMirror 6 Extensions:** שימוש בתוספים הרשמיים ל-Folding, Bracket Matching ו-Linting.
2.  **Local Storage:** שמירת מצב הטאבים, היסטוריית החיפוש והגדרות התצוגה בצד הלקוח לביצועים מהירים.
3.  **Client-Side Rendering (חלקי):** מעבר לטעינת תוכן קבצים ב-AJAX/Fetch במקום טעינת עמוד מלאה, כדי לאפשר את ממשק הטאבים החלק.
