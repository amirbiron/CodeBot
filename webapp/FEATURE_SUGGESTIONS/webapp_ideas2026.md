# 💡 רעיונות לפיצ'רים ל־WebApp (2026)

תאריך: 2026-01-23  
מטרה: רעיונות פרקטיים שמועילים למשתמשים, “קשה לסרב להם”.  
לא כולל: פיצ'רי קהילה/סושיאל, AI Agents.  

## מה נסרק כדי להימנע מכפילויות
- קבצי UI מרכזיים: `webapp/templates/*` (כולל `files.html`, `view_file.html`, `edit_file.html`, `dashboard.html`)  
- JS מרכזי: `webapp/static/js/*` (למשל `editor-manager.js`, `file-form-manager.js`, `live-preview.js`, `repo-browser.js`, `card-preview.js`)  
- Build assets: `webapp/static_build/*` + PWA (`webapp/static/sw.js`, `webapp/static/manifest.json`)  
- מסמכי רעיונות קיימים: `webapp/FEATURE_SUGGESTIONS/` (כולל Global Search / Multi-select / Compare / CodeMirror / Themes ועוד)

> הערה: לא קיים בפועל קובץ `webapp/**/editor.js` וגם לא קיימת תיקיית `webapp/config/` — בפועל הלוגיקה מפוזרת תחת `webapp/static/js/`, תבניות Jinja ו־`static_build/`.

## הצעות פיצ'רים (2026)

### 🖼️ Drag & Drop תמונות לתוך Markdown + הזרקת קישור אוטומטית
תיאור: גרירה של תמונה לתוך העורך תעלה אותה (כמו ה־widget הקיים) וגם תכניס אוטומטית שורת Markdown מתאימה במקום הסמן. זה חוסך “העליתי אבל שכחתי לקשר”.
קטגוריה: Editor / Markdown
מורכבות: בינונית

### 🧷 “נעץ לדשבורד” ישירות מרשימת הקבצים
תיאור: כפתור נעיצה קטן בכל כרטיס קובץ (ליד “צפה/הורד”) כדי שלא צריך להיכנס לדף הקובץ רק בשביל לנעוץ.
קטגוריה: Dashboard / Productivity
מורכבות: קלה

### 🗂️ סידור נעוצים: גרירה לשינוי סדר + קבוצות
תיאור: אפשרות לגרור כרטיסים נעוצים כדי לקבוע סדר, וליצור “קבוצות” (למשל “פרויקט X”, “תחזוקה”, “דוקומנטציה”) כדי שהדשבורד יישאר נקי.
קטגוריה: Dashboard / Organization
מורכבות: בינונית

### 🧾 “תצוגת גרסאות” חכמה: Diff קטן בתוך מודל ההיסטוריה
תיאור: בתוך “היסטוריית גרסאות” להוסיף מצב “הצץ להבדלים” (למשל מול הגרסה הקודמת) בלי לעבור למסך אחר.
קטגוריה: Versioning / UX
מורכבות: בינונית

### 🧰 מרכז ייצוא (Export Center) עם שמות, תוקף וחידוש
תיאור: במקום לראות ZIPים בלבד, לנהל אותם: שינוי שם ידידותי, תיאור, מה כלול, הורדה מחדש, מחיקה, ותאריך תפוגה ברור.
קטגוריה: Export / Backup
מורכבות: בינונית

### 🧹 שינוי שם קבוצתי (Bulk Rename) עם Preview
תיאור: בבחירה מרובה להוסיף “שנה שמות”: prefix/suffix, החלפה טקסטואלית, ואפילו Regex — עם תצוגה מקדימה לפני שמירה.
קטגוריה: Productivity / Files
מורכבות: מורכב

### 🏷️ מנהל תגיות (Tag Manager): איחוד/שינוי שם/מחיקה + ספירות
תיאור: מסך שמציג את כל התגיות עם ספירה, מאפשר לאחד תגיות כפולות (למשל `utils` ו־`utility`) ולשנות שם בצורה בטוחה.
קטגוריה: Organization
מורכבות: בינונית

### 🧭 Quick Switcher לקבצים (Ctrl/Cmd+P)
תיאור: חלון קטן שמופיע בכל מקום ומאפשר לקפוץ לקובץ לפי שם, כולל pinned/recent, עם מקלדת בלבד.
קטגוריה: Navigation / Productivity
מורכבות: בינונית

### 📌 “קיצורי מקלדת” — Overlay לפי עמוד
תיאור: תפריט עזרה קצר שמציג קיצורים רלוונטיים לעמוד (Files / View / Edit / Repo Browser), כולל אפשרות להעתקה כטקסט.
קטגוריה: UX / Accessibility
מורכבות: קלה

### 📑 “טאבים” של קבצים אחרונים (בצפייה/עריכה)
תיאור: פס קטן שמציג את 5–8 הקבצים האחרונים שנפתחו, כדי לעבור ביניהם מהר בלי לחפש מחדש.
קטגוריה: Navigation
מורכבות: בינונית

### 🧷 פעולות “העתק כ־…” (Copy As…)
תיאור: הרחבת כפתור ההעתקה ל־3 מצבים: Raw, “Markdown fenced” עם שפה, ו”קובץ עם Header” (שם/שפה/תגיות) לשיתוף נוח.
קטגוריה: Sharing / Productivity
מורכבות: קלה

### 🧱 מצב “קובץ ענק” משודרג (טעינה הדרגתית + קפיצה לשורה)
תיאור: לקבצים שמסומנים כגדולים — להציג “חלונות” של טווח שורות, חיפוש מהיר, וקפיצה לשורה בלי לטעון הכל בבת אחת.
קטגוריה: Performance / Viewer
מורכבות: מורכב

### 🧾 “בטל פעולה” לפעולות מסוכנות (Undo Window)
תיאור: אחרי פעולות כמו Bulk Delete/Tag/Move — להציג טוסט עם “בטל” למשך 10–30 שניות, כדי להוריד פחד מטעויות.
קטגוריה: UX / Safety
מורכבות: בינונית

### 🔐 רשימת קישורי שיתוף פעילים + ביטול בלחיצה
תיאור: מסך שמרכז את כל הקישורים שנוצרו (זמני/קבוע), מציג מה פתוח עדיין, ומאפשר לבטל קישור בלי לחפש איפה נוצר.
קטגוריה: Sharing / Security
מורכבות: בינונית

### 🧭 חזרה חכמה ל”תוצאות האחרונות” (Back to Results)
תיאור: כפתור “חזרה לרשימה” שמחזיר בדיוק למסננים/עמוד/גלילה האחרונים (ב־Files וגם בדפדפן הקוד).
קטגוריה: UX
מורכבות: קלה

### 🧩 “עריכה מהירה” של תיאור/תגיות גם ברשימת הקבצים
תיאור: כמו מודל העריכה המהירה שקיים ב”קבצים שדורשים טיפול” — אבל זמין לכל כרטיס קובץ, בלי לצאת מהרשימה.
קטגוריה: Productivity / Metadata
מורכבות: בינונית

### 🧪 בדיקת “סודות בטקסט” לפני שיתוף/ייצוא
תיאור: זיהוי פשוט (Regex) של טוקנים נפוצים (API keys, JWT, private keys) והצגת אזהרה לפני יצירת קישור שיתוף/ZIP.
קטגוריה: Security
מורכבות: בינונית

### 🧭 אחידות התראות: Toasts במקום `alert()` בכל האתר
תיאור: כיום יש מקומות שנופלים ל־`alert`. רכיב Toast אחיד ייצור חוויה מקצועית ויחסוך “קפיצות” שמפריעות לזרימה.
קטגוריה: UX / Polish
מורכבות: קלה

### 🗓️ תצוגת “מה חדש” לפי גרסאות + סימון “נקרא”
תיאור: להפוך את “מה חדש” לרשימת Release Notes קצרה: פילטר לפי חודש/גרסה, וסימון “קראתי” כדי שלא יחזור.
קטגוריה: Dashboard / UX
מורכבות: בינונית

### 🧭 שילוב הדוק בין “Repo Browser” ל־“הקבצים שלי”
תיאור: כפתור “שמור לתיקייה שלי” מתוך Repo Browser (בצד הקובץ), וכפתור “פתח בדפדפן קוד” מתוך דף קובץ, כדי לחבר בין שני העולמות.
קטגוריה: Workflow
מורכבות: בינונית

### 🧾 תצוגת “שינויים מאז הפעם הקודמת” לקבצים נעוצים
תיאור: עבור קבצים נעוצים — להציג בדשבורד חיווי “עודכן מאז שצפית” + קפיצה לגרסה/היסטוריה.
קטגוריה: Dashboard / Productivity

### 🧭 “מצב עבודה מרוכז” לעריכה (Focus)

מורכבות: קלה

תיאור: בפתיחת “הוסף קובץ חדש” לבחור תבנית קצרה (README, Docker Compose, GitHub Actions, SQL migration) כדי להתחיל מהר בלי להעתיק מבחוץ.

מורכבות: בינונית
