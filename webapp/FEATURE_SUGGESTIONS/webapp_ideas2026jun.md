# 💡 WebApp Ideas — יוני 2026 (ללא כפילויות)
**תאריך:** יוני 2026  
**סקופ סריקה:** `webapp/` כולל `static/`, `static_build/`, `templates/`, וקבצי JS מרכזיים (למשל `editor-manager.js`, `live-preview.js`) + הצלבה מול `webapp/FEATURE_SUGGESTIONS/`.  
**מיקוד:** פיצ׳רים שמועילים למשתמשים ומשפרים שימושיות/תפוקתיות/אמינות.  
**לא כלול:** Social/Community features, AI Agents.  

> הערה: המסמך הזה נכתב כדי להוסיף רעיונות **שלא חוזרים** על `webapp_ideas2026.md`, `webapp_improvement_ideas.md`, `webapp_ideas3.md` ושאר מדריכי המימוש/UX שבתיקייה.

---

## 🧭 ניווט, הקשר ומצב עבודה

### 🧭 סרגל “חזרה/קדימה” קבוע (כמו IDE)
תיאור: כפתורי Back/Forward בתוך ה־UI (בנוסף לדפדפן) שמחזירים אותך לקובץ הקודם/הבא תוך שמירה על פילטרים, מיקום גלילה וטאבים פתוחים.
קטגוריה: Navigation
מורכבות: בינונית

### 🧷 “המשך מאיפה שעצרת” לכל קובץ
תיאור: שמירת מיקום גלילה/שורה אחרונה שנצפתה לכל קובץ, והצעה לחזרה מהירה (“המשך קריאה”) כשפותחים אותו שוב.
קטגוריה: Reading UX
מורכבות: בינונית

### 🧭 “קפוץ להיסטוריה” בעמוד קובץ (Breadcrumb של פעולות)
תיאור: רשימה קצרה בראש הקובץ: נצפה/נערך/שותף/הועבר לסל — עם קישורי קיצור לפעולות הרלוונטיות (למשל פתיחת היסטוריה, חזרה לעריכה).
קטגוריה: Navigation
מורכבות: בינונית

### 🧱 שמירת מצב “Split View” לפי משתמש
תיאור: כשמשתמש מפעיל Split View (למשל Live Preview), המערכת זוכרת את רוחבי הפאנלים ואת הטאב הפעיל ומחזירה אותם בפעם הבאה.
קטגוריה: UX
מורכבות: קלה

## ✍️ עריכה (Editor) ו־Markdown/HTML

### 🔗 Sync Scroll בין העורך לתצוגה המקדימה
תיאור: בספליט־ויו, גלילה בעורך תגלול גם את התצוגה (ולהפך) כדי לעבוד מהר על מסמכים ארוכים בלי “לאבד מקום”.
קטגוריה: Editor UX
מורכבות: בינונית

### 🧭 “Outline” למסמך Markdown (Sidebar לפי כותרות)
תיאור: סיידבר שמציג כותרות (H1/H2/H3) מהתוכן הנערך ומאפשר קפיצה מהירה — במיוחד שימושי לקבצי תיעוד ארוכים.
קטגוריה: Editor UX
מורכבות: בינונית

### 🧩 “Copy As…” (Markdown/HTML/Plain)
תיאור: כפתור אחד שמאפשר להעתיק את הבחירה/כל הקובץ כ־Plain text, כ־Markdown fenced block עם שפה, או כ־HTML מעוצב (כשיש).
קטגוריה: Productivity
מורכבות: קלה

### 🧯 Undo לפעולות טופס (ניקוי טיוטה/שחזור)
תיאור: אחרי “נקה טיוטה” או פעולות דומות, הצג Toast עם Undo לכמה שניות כדי למנוע אובדן עבודה בטעות.
קטגוריה: Reliability
מורכבות: קלה

### 🧪 Preview “מצב Sandbox” ל־HTML (מוגבל ומסומן)
תיאור: תצוגת HTML עם דגל ברור שזה sandbox, עם אפשרות לבחור “iframe בלבד” כדי להפחית הפתעות (סקריפטים/סטיילים חיצוניים) ולשמור על ביטחון.
קטגוריה: Safety
מורכבות: בינונית

## 👀 צפייה בקוד וקריאות

### 🎯 “Go to line” בעמוד צפייה
תיאור: שדה קטן שמקפיץ לשורה מסוימת בקוד (ושומר את זה ב־URL), כדי להגיע מהר למקום מדויק בקבצים ארוכים.
קטגוריה: Code Viewer
מורכבות: קלה

### 🧾 מצב “קריאה נקייה” לקוד (Reading Mode)
תיאור: כפתור שמסתיר כפתורים/מטא־דאטה ומשאיר רק כותרת + קוד, עם טיפוגרפיה נוחה ומרווחים לקריאה ממושכת.
קטגוריה: Reading UX
מורכבות: קלה

### 🖨️ הדפסה/Export “נקי” לעמוד צפייה
תיאור: “תצוגת הדפסה” מסודרת (CSS print) לקוד/Markdown כך שאפשר להדפיס או לשמור PDF בצורה קריאה בלי אלמנטים מיותרים.
קטגוריה: Export
מורכבות: קלה

### 🧭 “Find next” חכם עם שמירת חיפוש בין דפים
תיאור: זכירת מחרוזת החיפוש האחרונה והמשך ניווט לתוצאה הבאה גם אם עוברים בין “תצוגת קוד” ל־“תצוגת Markdown/HTML”.
קטגוריה: Code Viewer
מורכבות: בינונית

## 🗂️ ארגון, מטא־דאטה ואיכות תוכן

### ✏️ עריכת מטא־דאטה “במקום” בעמוד צפייה
תיאור: עריכת תיאור/תגיות/Source URL מתוך עמוד הקובץ בלי לעבור למסך עריכה מלא (עם שמירה מהירה).
קטגוריה: UX
מורכבות: בינונית

### 🧹 “בדיקת כפילויות” בעת שמירה (התראה עדינה)
תיאור: כששומרים קובץ חדש/גרסה, הצג התראה אם יש קבצים מאוד דומים (שם/תגיות/תוכן) כדי למנוע בלגן במאגר האישי.
קטגוריה: Organization
מורכבות: מורכבת

### 🏷️ “איכות תגיות” — ניקוי והאחדה חכמה
תיאור: כלי שמציע איחוד תגיות כפולות/דומות (למשל `repo:foo/bar` מול `Repo:foo/bar`) ומסדר רווחים/תחיליות.
קטגוריה: Organization
מורכבות: בינונית

### 🧾 תבנית “Front‑Matter” ל־Markdown (שדות נפוצים)
תיאור: למי שמנהל תיעוד: כפתור שמוסיף בלוק YAML קטן בראש הקובץ עם שדות נפוצים (title, tags, source, updated) בלי ללכת לאיבוד.
קטגוריה: Documentation
מורכבות: קלה

## 📦 יבוא/יצוא ועבודה עם קבצים

### 🧳 “Export מהיר” לקובץ יחיד עם בחירת פורמט
תיאור: הורדה כ־`.txt`/`.md`/`.json`/`.html` (לפי סוג) עם שם קובץ נכון, בלי ללחוץ “הורד” ואז לערוך ידנית.
קטגוריה: Export
מורכבות: קלה

### 🧾 “Import Wizard” ל־Paste מרובה
תיאור: מסך שמאפשר להדביק כמה קבצים ברצף (עם מפריד/כותרת לכל קובץ), לעשות להם פיצול אוטומטי ולהגדיר שפה/תגיות בבת אחת.
קטגוריה: Import
מורכבות: מורכבת

### 🧪 בדיקת “שם קובץ תקין” לפני שמירה
תיאור: ולידציה ידידותית שמזהירה על שמות בעייתיים (ריקים/ארוכים מדי/תווים בעייתיים), עם הצעה לתיקון אוטומטי.
קטגוריה: UX
מורכבות: קלה

## ♿ נגישות ומובייל (כולל Mini‑App)

### ⌨️ “מפת קיצורים” קופצת (Cheat Sheet)
תיאור: חלון קצר שמציג קיצורי מקלדת זמינים לפי מסך (צפייה/עריכה/חיפוש), כדי שמשתמשים ימצאו יכולות מהר.
קטגוריה: Accessibility
מורכבות: קלה

### 🧠 מצבי RTL/LTR חכמים בתוך אותו קובץ
תיאור: כשיש טקסט בעברית לצד קוד/בלוקים באנגלית, לשפר קריאות עם כיוון אוטומטי לפסקאות (בלי לשבור בלוקים של קוד).
קטגוריה: Internationalization
מורכבות: בינונית

### 📱 “מצב יד אחת” במובייל
תיאור: העברת פעולות נפוצות (חיפוש, העתקה, מועדפים, שיתוף) לסרגל תחתון נגיש לאגודל, עם התנהגות טובה ב־Telegram Mini App.
קטגוריה: Mobile UX
מורכבות: בינונית

## 🔐 פרטיות, אמינות ושקט נפשי

### 🕶️ “מצב פרטיות” בעמוד צפייה (הסתרה מיידית)
תיאור: כפתור שמטשטש/מסתיר את התוכן על המסך עד לחיצה נוספת (למשל כשמישהו מסתכל), בלי לשנות הרשאות או לשמור מצב בשרת.
קטגוריה: Privacy
מורכבות: קלה

### 🧯 “אזור בטוח” לפעולות הרסניות
תיאור: ריכוז פעולות כמו “העבר לסל” בתוך תפריט ברור, עם אישור חכם (הקלדה קצרה/החזקה) כדי להפחית טעויות.
קטגוריה: Safety
מורכבות: בינונית

### 🧾 תצוגת “מה ישותף” לפני שיתוף
תיאור: לפני יצירת קישור שיתוף, הצג Preview קצר: שם קובץ, תיאור, תגיות, האם זה גרסה גדולה/קטנה — כדי למנוע שיתוף לא מכוון.
קטגוריה: Sharing
מורכבות: קלה

## 🧩 איכות חיים קטנה שעושה הבדל

### 🔎 חיפוש “מהיר” בכותרות/תגיות מתוך כל דף
תיאור: שדה קטן קבוע (Header) שמאפשר להתחיל להקליד ולקבל תוצאות מהירות לקבצים לפי שם/תגיות, בלי לעבור למסך חיפוש נפרד.
קטגוריה: Search
מורכבות: בינונית

### 📌 “פעולות אחרונות” (Last Actions) בדשבורד
תיאור: רשימה קצרה של הפעולות האחרונות (נפתח/נערך/שותף/יוצא) עם קיצור דרך חזרה לאותו מסך.
קטגוריה: Dashboard
מורכבות: קלה

### 📊 מחוון שימוש באחסון (Quota/Usage Bar)
תיאור: הצגת כמה מקום/כמה קבצים נשמרו, ואיפה “נשפך” הכי הרבה (קבצים גדולים/הרבה גרסאות), כדי לעזור למשתמשים להתנהל.
קטגוריה: Analytics
מורכבות: בינונית
