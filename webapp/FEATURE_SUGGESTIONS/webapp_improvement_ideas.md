## צפיין קוד מתקדם

### 1) Diff Viewer – השוואת גרסאות

**למה:**  
משתמשים רוצים לראות מה השתנה בין גרסה לגרסה של קובץ. כרגע יש היסטוריית גרסאות אבל אין תצוגת הבדלים.

**מה מוסיפים:**
- תצוגת diff בסגנון GitHub (side-by-side או unified)
- השוואה בין כל שתי גרסאות
- הדגשת שינויים (הוספות/מחיקות/שינויים)
- אפשרות להחזיר גרסה ישנה

**איך בקצרה:**
- קריאת שתי גרסאות מ-MongoDB
- חישוב diff בצד שרת עם `difflib` (Python)
- רינדור עם סימון צבעוני (ירוק/אדום)
- כפתור "השווה גרסאות" בעמוד view_file

**אימפקט/מאמץ:** גבוה/בינוני

---

### 2) Code Folding – קיפול קטעים

**למה:**  
בקבצים ארוכים, רוצים להסתיר קטעי קוד שלא רלוונטיים כרגע כדי להתמקד בחלק מסוים.

**מה מוסיפים:**
- קיפול אוטומטי של פונקציות/מחלקות/בלוקים
- אייקון קטן ליד מספר שורה לקיפול/פתיחה
- זכירת מצב קיפול ב-sessionStorage
- קיצור מקלדת: `Ctrl+[` לקיפול, `Ctrl+]` לפתיחה

**איך בקצרה:**
- זיהוי בלוקים לפי indentation או סימני `{}`, `def`, `class`
- CSS לעטיפה עם `...` מקוקד
- JS פשוט להוספת/הסרת class `.folded`

**אימפקט/מאמץ:** גבוה/נמוך

---

### 3) Split View – צפייה בשני קבצים בו-זמנית

**למה:**  
לעיתים צריך להשוות או להעתיק קוד בין שני קבצים. כרגע צריך לפתוח כרטיסיות נפרדות.

**מה מוסיפים:**
- מצב split אופקי או אנכי
- כפתור "פתח ב-split" בעמוד הקובץ
- סנכרון גלילה (אופציונלי)
- גרירה לשינוי גודל הפאנלים

**איך בקצרה:**
- פריסת CSS Grid או Flexbox עם שני מיכלי קוד
- כפתור בheader להפעלת מצב split
- בוחרים קובץ שני מרשימה מהירה
- state management ב-sessionStorage

**אימפקט/מאמץ:** בינוני/בינוני

---

### 4) Syntax Validation – בדיקת שגיאות בזמן אמת

**למה:**  
טעויות תחביר נפוצות (סוגריים חסרים, הזחה שגויה בPython) קשה לזהות בלי IDE. הוספת אזהרות תסייע.

**מה מוסיפים:**
- בדיקה אוטומטית כש-מעלים או עורכים קובץ
- סימון שורות עם שגיאות (קו אדום מתחת)
- הודעת שגיאה בצד (tooltip)
- תמיכה ב-Python, JS, JSON, YAML

**איך בקצרה:**
- שילוב של linters: `pylint`/`pyflakes` (Python), `eslint` (JS), `yamllint`
- הרצה בbackground worker (Celery/RQ)
- החזרת JSON עם שורות שגיאה
- הצגה ב-UI עם אייקון אזהרה

**אימפקט/מאמץ:** גבוה/בינוני

---

### 5) Code Minimap – מפת ניווט מצומצמת

**למה:**  
בקבצים ארוכים, קשה לנווט ולהבין את המבנה. minimap נותן תמונה כוללת.

**מה מוסיפים:**
- פס צר בצד המסך עם תצוגה מוקטנת של הקוד
- סימון מיקום נוכחי
- לחיצה על ה-minimap קופצת לאזור
- סימון חזותי של סימניות וחיפושים

**איך בקצרה:**
- רינדור של הקוד בגודל זעיר (1-2px לשורה)
- Canvas או SVG לביצועים
- scroll event listener לעדכון מיקום
- CSS לעיצוב מינימליסטי

**אימפקט/מאמץ:** בינוני/בינוני

## נוחות וזרימת עבודה

### 6) Quick Access Panel – פאנל גישה מהירה

**למה:**  
צריך לעבור בין קבצים שונים הרבה. כרגע צריך לחזור לרשימת קבצים בכל פעם.

**מה מוסיפים:**
- פאנל צד קטן עם:
  - קבצים שנפתחו לאחרונה (5-10)
  - קבצים מועדפים
  - קבצים פתוחים (tabs)
- קיצור: `Ctrl+P` לחיפוש מהיר והקפצה לקובץ
- גרירת קבצים בין קטגוריות

**איך בקצרה:**
- שמירה ב-localStorage + sync עם שרת
- רשימה דינמית ב-sidebar או פאנל צף
- search box עם autocomplete
- עדכון אוטומטי בכל פתיחת קובץ

**אימפקט/מאמץ:** גבוה/נמוך

---

### 7) Version History Timeline – ציר זמן גרסאות

**למה:**  
כרגע רואים רשימת גרסאות, אבל קשה להבין מתי ומה השתנה בפועל.

**מה מוסיפים:**
- תצוגת timeline חזותית
- כל גרסה עם תאריך, גודל שינוי, סיכום אוטומטי
- אפשרות לצפות/להשוות/לשחזר
- אייקונים ויזואליים (שעון, גרף שינויים)

**איך בקצרה:**
- שאילתת MongoDB למיון לפי `updated_at`
- חישוב diff size בין גרסאות
- רינדור עם CSS timeline (קו אנכי + נקודות)
- modal או עמוד ייעודי לכל גרסה

**אימפקט/מאמץ:** גבוה/בינוני

---

### 8) Smart Templates – תבניות קוד מוכנות

**למה:**  
משתמשים כותבים לעיתים קרובות אותן מבנים (class Python, function JS, HTML boilerplate). חיסכון בזמן.

**מה מוסיפים:**
- ספריית תבניות מובנית ומותאמת אישית
- כפתור "צור מתבנית" בעת יצירת קובץ חדש
- משתנים דינמיים: `{{CLASS_NAME}}`, `{{DATE}}`
- שיתוף תבניות בין משתמשים (אופציונלי)

**איך בקצרה:**
- MongoDB collection: `templates`
- טופס הוספת תבנית עם שם, שפה, תוכן
- בעת יצירה: בחירה מרשימה + החלפת משתנים
- אפשרות ייבוא/ייצוא JSON

**אימפקט/מאמץ:** בינוני/נמוך

---

### 9) Code Snippets Library – ספריית קטעי קוד לשימוש חוזר

**למה:**  
לעיתים יש קטעי קוד קטנים (utils, configs) שרוצים להעתיק מהר. כרגע צריך לחפש בקובץ מלא.

**מה מוסיפים:**
- אפשרות לסמן קטע קוד כ-"snippet" (חלק מקובץ)
- ספרייה ייעודית לכל ה-snippets
- חיפוש לפי שפה/תגיות
- העתקה מהירה בלחיצה אחת

**איך בקצרה:**
- שמירת שורות התחלה/סיום + קוד במבנה:
  ```json
  {
    "file_id": "...",
    "start_line": 10,
    "end_line": 25,
    "code": "...",
    "tags": ["utils", "string"]
  }
  ```
- עמוד `/snippets` עם גריד
- כפתור "שמור כsnippet" בצד הקוד

**אימפקט/מאמץ:** גבוה/בינוני

---

### 10) Keyboard Shortcuts Hub – מרכז קיצורי מקלדת

**למה:**  
יש כבר כמה קיצורים (Ctrl+B לסימניות, Ctrl+A, Escape), אבל לא ברור למשתמש מה קיים.

**מה מוסיפים:**
- מסך עזרה: `?` או `Ctrl+/`
- רשימה מלאה של כל הקיצורים
- אפשרות התאמה אישית (אופציונלי)
- קיצורים מחולקים לקטגוריות (ניווט, עריכה, חיפוש)

**איך בקצרה:**
- modal או עמוד ייעודי עם טבלה
- bind של `?` לפתיחת ה-modal
- שמירת קיצורים מותאמים ב-localStorage
- עיצוב מינימליסטי בסגנון הקיים

**אימפקט/מאמץ:** גבוה/נמוך

---

### 11) Export/Import מלא – גיבוי ושחזור

**למה:**  
משתמשים רוצים לייצא את כל הקבצים שלהם (למשל לגיבוי או מעבר מערכת).

**מה מוסיפים:**
- ייצוא כ-ZIP עם כל הקבצים + metadata (JSON)
- ייבוא מ-ZIP חזרה למערכת
- בחירת קבצים ספציפיים או הכל
- שמירת מבנה תיקיות (לפי תגיות/ריפו)

**איך בקצרה:**
- endpoint: `/api/export/all` מחזיר ZIP stream
- בנייה עם `zipfile` (Python)
- metadata בקובץ `manifest.json`
- ייבוא: `/upload` עם parse של ZIP

**אימפקט/מאמץ:** גבוה/בינוני

---

### 12) Activity Feed – עדכוני פעילות בזמן אמת

**למה:**  
מעניין לראות מה עשיתי לאחרונה: קבצים שנוצרו, נערכו, נמחקו.

**מה מוסיפים:**
- רשימת פעולות אחרונות בדשבורד
- כל פעולה עם תאריך, אייקון, קישור
- פילטר לפי סוג פעולה (create, edit, delete, share)
- אופציה לשחזור פעולות (undo)

**איך בקצרה:**
- לוג כל פעולה ב-collection: `activity_log`
- שאילתת 20 אחרונות לפי `user_id`
- רינדור ב-timeline או כרטיסים
- כפתור undo לפעולות מסוימות (delete)

**אימפקט/מאמץ:** בינוני/נמוך-בינוני

## שיתוף פעולה והערות

### 13) Comments & Annotations – הערות על קטעי קוד

**למה:**  
לעיתים רוצים לכתוב הערה על קטע קוד מסוים (לא רק שורה יחידה). שיתוף עם אחרים או תיעוד עצמי.

**מה מוסיפים:**
- אפשרות לסמן טווח שורות והוספת הערה
- הערות נשמרות לפי קובץ ומשתמש
- תצוגה ב-sidebar או tooltip
- תמיכה בMarkdown בהערות

**איך בקצרה:**
- collection: `annotations`
  ```json
  {
    "file_id": "...",
    "user_id": "...",
    "start_line": 10,
    "end_line": 15,
    "text": "זה צריך שיפור...",
    "created_at": "..."
  }
  ```
- הדגשת הטווח בצהוב בהיר
- אייקון הערה בצד
- modal לעריכה/מחיקה

**אימפקט/מאמץ:** גבוה/בינוני

### 14) Real-time Collaboration – עבודה משותפת בזמן אמת

**למה:**  
עבודה בצוות – כמה אנשים רוצים לערוך או לצפות בקובץ בו-זמנית.

**מה מוסיפים:**
- עריכה משותפת בזמן אמת (כמו Google Docs)
- סמנים צבעוניים של משתמשים אחרים
- chat צדדי או הערות inline
- היסטוריית שינויים ב-realtime

**איך בקצרה:**
- WebSocket (Flask-SocketIO) או polling
- Operational Transform או CRDT לסנכרון
- MongoDB אחסון מצב ביניים
- UI עם cursors וסמנים

**אימפקט/מאמץ:** גבוה/גבוה

---

### 15) Shared Workspaces – מרחבי עבודה משותפים

**למה:**  
לעבוד בצוות על סט קבצים משותף (לא רק שיתוף קישורים).

**מה מוסיפים:**
- יצירת "workspace" עם הרשאות
- הוספת משתמשים לעבודה משותפת
- ניהול הרשאות (read/write/admin)
- תיקיות משותפות לארגון

**איך בקצרה:**
- collection: `workspaces`
- שדה `members: [{user_id, role}]`
- פילטר בדפים: "הקבצים שלי" + "workspaces"
- API endpoints לניהול

**אימפקט/מאמץ:** גבוה/גבוה

## כלי פרודוקטיביות נוספים

### 16) Code Playground – מגרש משחקים לניסוי

**למה:**  
לעיתים רוצים לנסות קוד במהירות בלי לשמור קובץ חדש.

**מה מוסיפים:**
- עורך זמני עם הרצה בדפדפן
- תמיכה ב-HTML/CSS/JS, Python (בשרת עם sandbox)
- תוצאות בזמן אמת
- שמירה כקובץ חדש אם רוצים

**איך בקצרה:**
- עמוד `/playground`
- iframe או sandboxed execution
- Python: הרצה ב-subprocess מוגבל (timeout, memory)
- JS: הרצה ישירה בדפדפן

**אימפקט/מאמץ:** בינוני/בינוני

### 17) Code Formatter – עיצוב אוטומטי

**למה:**  
קוד לא מסודר קשה לקריאה. עיצוב אוטומטי חוסך זמן.

**מה מוסיפים:**
- כפתור "עצב קוד" בעמוד עריכה
- תמיכה ב-Python (black/autopep8), JS (prettier), HTML
- preview לפני שמירה
- הגדרות סגנון (2/4 spaces, quotes)

**איך בקצרה:**
- integration עם formatters דרך subprocess
- endpoint: `/api/format` מקבל קוד ומחזיר מעוצב
- כפתור בעמוד edit
- תצוגת diff אופציונלית

**אימפקט/מאמץ:** בינוני/נמוך-בינוני

---

### 18) Smart Tag Suggestions – הצעות תגיות חכמות

**למה:**  
תיוג קבצים חשוב לארגון, אבל לעיתים שוכחים או לא יודעים איזה תגיות להוסיף.

**מה מוסיפים:**
- ניתוח תוכן הקובץ והצעת תגיות אוטומטיות
- למשל: קוד עם `@app.route` → `#flask`
- למידה מהיסטוריה: תגיות שהמשתמש משתמש בהן
- autocomplete מתקדם

**איך בקצרה:**
- regex patterns לזיהוי פרמטרים נפוצים
- ניתוח שפה (NLP בסיסי) או keywords
- שמירת תגיות פופולריות ב-cache
- הצגת הצעות בעת יצירה/עריכה

**אימפקט/מאמץ:** בינוני/בינוני

### 19) Dark/Light Theme Toggle – מעבר בין מצבים

**למה:**  
יש משתמשים שמעדיפים מצב כהה, אחרים בהיר. כרגע יש רק gradient סגול.

**מה מוסיפים:**
- כפתור במהיר בheader או הגדרות
- 3 מצבים: Auto (לפי מערכת), Light, Dark
- שמירת העדפה ב-localStorage + MongoDB
- CSS variables לצבעים

**איך בקצרה:**
- CSS: שתי ערכות צבעים (`:root`, `.dark-mode`)
- JS: toggle class על `<body>`
- API לשמירת העדפה
- respect של `prefers-color-scheme`

**אימפקט/מאמץ:** גבוה/נמוך-בינוני

---

### 20) File Preview Cards – תצוגה מקדימה מהירה

**למה:**  
בעמוד הקבצים, רוצים לראות מבט קצר על התוכן בלי לפתוח.

**מה מוסיפים:**
- hover על כרטיס קובץ מציג popup עם:
  - 5-10 שורות ראשונות
  - מספר שורות כולל
  - תאריך עדכון אחרון
  - תגיות
- אנימציה חלקה

**איך בקצרה:**
- שליפת snippet בשאילתה (או מ-cache)
- CSS: `.file-card:hover + .preview-popup`
- position absolute עם z-index גבוה
- debounce על hover למניעת flickering

**אימפקט/מאמץ:** בינוני/נמוך
