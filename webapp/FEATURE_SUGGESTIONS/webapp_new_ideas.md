# רעיונות חדשים ל־WebApp (ממוקדי ערך למשתמש)

מסמך זה נכתב אחרי סקירה של `webapp/` (כולל `templates/`, `static/`, `static_build/`) וגם סקירה של המסמכים הקיימים ב־`webapp/FEATURE_SUGGESTIONS/` כדי **לא לחזור על רעיונות שכבר הוצעו** (למשל: CodeMirror, Floating TOC, Config Radar, Incident Replay, Focus/Power modes, Global Search, Compare).

> הערה טכנית: לא נמצא `webapp/config/` ולא נמצא `editor.js` בנתיב הזה; בפועל יש קבצי editor תחת `webapp/static/js/` (למשל `editor-manager.js`).

מה לא כלול בכוונה: פיצ׳רי קהילה/שיתופים חברתיים, וסוכן AI לבוט.

---

### 🗂️ “Saved Views” לקבצים (מסכים חכמים)
**תיאור:** אפשרות לשמור קומבינציה של פילטרים/מיון/חיפוש בעמוד `files` כ״תצוגה״, ולפתוח אותה בקליק (למשל: “Markdown אחרונים”, “קבצים עם תגית repo:X”).  
**קטגוריה:** UI  
**מורכבות:** בינונית

### 🧷 אינדקס פתקים + חיפוש פתקים בתוך קובץ
**תיאור:** פאנל צד קטן בעמוד Markdown שמציג רשימת פתקים בקובץ (כותרת/שורה/עודכן לאחרונה) עם חיפוש מהיר וקפיצה לפתק.  
**קטגוריה:** UI  
**מורכבות:** בינונית

### 🧲 “מצבי פתק” ברורים: צף / נעוץ / מעוגן לשורה/כותרת
**תיאור:** להפוך את היכולות שכבר קיימות (נעיצה/עוגן) למצב UI ברור עם מתג: פתק צף (fixed), פתק נעוץ למיקום בדף (absolute), או פתק שמוצמד לשורה/כותרת (ומזיז את עצמו בגלילה).  
**קטגוריה:** UI  
**מורכבות:** בינונית

### 🗃️ קיבוץ פתקים לפי פרקים (Headings) + “פתקים פרקיים”
**תיאור:** בעמוד Markdown להציג “פתקים של הפרק הזה” ליד כל כותרת H2/H3, כדי להפוך את המסמך למחברת עבודה ולא רק צפייה.  
**קטגוריה:** UI  
**מורכבות:** מורכבת

### 🔁 ייצוא פתקים לתוך הקובץ (כבלוק Markdown אוטומטי)
**תיאור:** כפתור “ייצא פתקים” שיוצר בלוק מסודר בסוף הקובץ (או בקובץ חדש) עם כל הפתקים, כולל קישורים לקפיצה (לכותרת/שורה).  
**קטגוריה:** Dev Tool  
**מורכבות:** בינונית

### ⏰ תצוגת “פתקים עם תזכורת” מרוכזת
**תיאור:** דף/ווידג׳ט שמציג רק פתקים שיש להם תזכורת (מה שממש עוזר לנהל עבודה), עם פילטרים: היום/השבוע/באיחור.  
**קטגוריה:** UI  
**מורכבות:** בינונית

### 📌 קישור שיתוף ישיר לפתק (Deep Link)
**תיאור:** לכל פתק להוסיף “העתק קישור” שפותח את אותו קובץ/Markdown ומקפיץ אותך לפתק הספציפי (באמצעות `?note_id=` או hash).  
**קטגוריה:** UI  
**מורכבות:** קלה

### 🧭 “Jump Bar” ל־Markdown: חזרה למיקום אחרון + נקודות עניין
**תיאור:** סרגל קטן צף שמציג: חזרה למיקום האחרון שקראת, קפיצה לכותרת הקרובה, וקפיצה ל”פתק הבא/הקודם”.  
**קטגוריה:** UI  
**מורכבות:** בינונית

### 🧼 “Draft Recovery” חכם לעריכה (שכבת התאוששות אמיתית)
**תיאור:** בנוסף לשמירה רגילה, להציג באנר “נמצאה טיוטה מקומית חדשה יותר” עם אפשרויות: שחזר / השווה / מחק, כדי לא לאבד תוכן אחרי רענון/קריסה.  
**קטגוריה:** Dev Tool  
**מורכבות:** בינונית

### 🧾 “Copy as Snippet” אחיד בכל המערכת
**תיאור:** כפתור העתקה אחד שמייצר פורמט קבוע: שם קובץ + קישור + שורות (אם יש) + בלוק קוד Markdown, כדי להדביק בקלות לתיעוד/טיקטים.  
**קטגוריה:** Dev Tool  
**מורכבות:** קלה

### 🧭 “Breadcrumbs” לפי Repo/Path מתוך תגיות
**תיאור:** אם קיימות תגיות `repo:` ו־`path:` — להציג למעלה “פירורי לחם” שמאפשרים פילטור מיידי לקבצים מאותו repo/תיקייה.  
**קטגוריה:** UI  
**מורכבות:** קלה

### 🧷 פעולות מהירות בתוצאות חיפוש גלובלי
**תיאור:** בתוך כרטיס תוצאת חיפוש להוסיף פעולות בלי לעבור דף: “פתח”, “העתק קטע”, “הוסף למועדפים”, “הוסף לאוסף”.  
**קטגוריה:** UI  
**מורכבות:** בינונית

### 🧱 “Result Preview Drawer” לתוצאות חיפוש
**תיאור:** במקום לפתוח טאב חדש לכל תוצאה — מגירה צדדית שמציגה Preview של הקובץ סביב ה־snippet, עם כפתורי מעבר לתוצאה הבאה/הקודמת.  
**קטגוריה:** UI  
**מורכבות:** מורכבת

### 🧩 “Apply Diff as New Version” (מהשוואה לגרסה חדשה)
**תיאור:** אחרי שמייצרים diff ב־Compare, לאפשר “החל את השינויים כגרסה חדשה” (עם תצוגת אישור), כדי להפוך diff לכלי עבודה ולא רק צפייה.  
**קטגוריה:** Dev Tool  
**מורכבות:** מורכבת

### 🧵 “Section Navigator” בתוך Diff ארוך
**תיאור:** רשימת ניווט צדדית שמקבצת את ה־diff לפי בלוקים/אזורים ומאפשרת קפיצה בין שינויים, במיוחד ב־inline/unified.  
**קטגוריה:** Visualization  
**מורכבות:** בינונית

### 🧯 “Bug Report Bundle” למשתמשים (Export מצב תקלה)
**תיאור:** כפתור שמייצא חבילת JSON: URL, פילטרים פעילים, שגיאות JS אחרונות, נתוני ביצועים בסיסיים, וטיימינג של קריאות API — כדי לדווח תקלות מהר.  
**קטגוריה:** Dev Tool  
**מורכבות:** בינונית

### 🧭 “Saved Observability Views” (טווח+פילטרים+דפים)
**תיאור:** בלוח Observability לשמור סט פילטרים (timerange, severity, endpoint, חיפוש) ולחזור אליו מהר, כולל שיתוף קישור “קבוע” לצוות.  
**קטגוריה:** Observability  
**מורכבות:** בינונית

### 🧰 “Runbook Studio” לניהול Runbooks (מיפוי לפי Alert Type/Endpoint)
**תיאור:** מסך אדמין שמאפשר להגדיר/לערוך Runbooks בצורה ידידותית: צעדים, פעולות Quick Fix מוצמדות, ותנאי הצגה לפי `alert_type`/`endpoint`.  
**קטגוריה:** Admin  
**מורכבות:** מורכבת

### 🧾 “Postmortem Pack” בלחיצה אחת
**תיאור:** ביצוא מאירוע (Story) לייצר “חבילת Postmortem” (Markdown/ZIP): סיפור, טבלה של התראות קשורות, גרפים מרכזיים, ורשימת פעולות שבוצעו (Quick Fix/Runbook).  
**קטגוריה:** Observability  
**מורכבות:** מורכבת

### 🔗 “From Alert to Code” (קישור חכם מחומרת תקלה לקבצים)
**תיאור:** כשב־metadata יש `file`/`endpoint`/`repo` — להציע כפתור “חפש בקבצים” שמריץ חיפוש גלובלי/פילטרים מוכנים כדי להגיע מהר לקוד הרלוונטי.  
**קטגוריה:** Observability  
**מורכבות:** בינונית

### 🧭 “Config Inspector: Diff + Validate” לפני שמירה
**תיאור:** בעמוד Admin של Config Inspector להציג diff לפני שמירה + ולידציה (JSON/YAML) עם שגיאות ברורות, כדי למנוע “שברנו פרודקשן בגלל פסיק”.  
**קטגוריה:** Admin  
**מורכבות:** בינונית

### 🧾 “Theme Snapshot Export/Import”
**תיאור:** אפשרות לייצא/לייבא סט הגדרות UI (ערכת נושא, מצב תצוגה, העדפות עורך וכו׳) כ־JSON כדי לשכפל הגדרות בין מכשירים.  
**קטגוריה:** Admin  
**מורכבות:** בינונית
